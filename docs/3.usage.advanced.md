# PromptLens — 확장 기능 가이드

> 프로젝트 관리, 버전 관리, 시각화, API 모드, 로컬 파일 저장소의 상세 사용법입니다.
> 기본 설치와 첫 분석은 [README.md](../README.md)를 먼저 확인하세요.

## Step 4. 프로젝트 만들기

프롬프트 분석 기록을 프로젝트별로 관리할 수 있습니다.

```
PromptLens에 "웹 개발" 프로젝트를 만들어줘
```

프로젝트를 만든 후, 분석할 때 프로젝트에 저장할 수 있습니다:

```
"REST API 설계해줘"를 분석하고 "웹 개발" 프로젝트에 저장해줘
```

나중에 기록을 확인하려면:

```
"웹 개발" 프로젝트의 히스토리를 보여줘
```

### 활성 프로젝트 설정 — 프로젝트 이름 매번 지정 생략

프로젝트를 한 번 활성으로 설정해두면 이후 `>> 분석` 시 자동으로 해당 프로젝트에 저장되고 **entryId가 항상 발급**됩니다.

```
"웹 개발"을 활성 프로젝트로 설정해줘
```

이후부터는 프로젝트 이름 없이 분석해도 자동 저장됩니다:

```
React로 로그인 폼 만들어줘 >> anz
→ 📊 43점 (D등급) ... entryId: h_xxxx  ← 항상 발급됨
```

| 명령 | 예시 |
|------|------|
| 활성 프로젝트 설정 | `"'웹 개발'을 활성 프로젝트로 설정해줘"` |
| 활성 프로젝트 확인 | `"지금 활성 프로젝트가 뭐야?"` |
| 활성 프로젝트 해제 | `"활성 프로젝트 해제해줘"` |

> 활성 프로젝트는 `~/.promptlens/settings.json`에 저장되며 Claude Desktop을 재시작해도 유지됩니다.

## Step 4-B. 로컬 파일 저장소 (v0.4.0)

프로젝트 데이터를 로컬 파일로 내보내고, 가져오고, 고급 검색할 수 있습니다.

### 프로젝트 내보내기 & 공유

```
"웹 개발" 프로젝트를 JSON으로 내보내줘
```

→ `~/.promptlens/exports/웹개발-20260324.promptlens.json` 생성. 팀원에게 공유하거나 다른 컴퓨터로 이동할 때 활용합니다.

형식별 용도:

| 형식 | 활용 |
|------|------|
| `json` | 팀 공유, 다른 머신 복원 (`import_project`로 재가져오기 가능) |
| `markdown` | 주간 리포트, PR 첨부 |
| `csv` | Excel/Google Sheets 분석 |

```
A등급 항목만 Markdown으로 내보내줘 (grade: A)
이번 달 분석 결과를 CSV로 내보내줘
```

### 프로젝트 가져오기

다른 머신이나 팀원에게 받은 `.promptlens.json` 파일을 가져옵니다:

```
backup.promptlens.json을 가져와줘
backup.promptlens.json을 "웹 개발" 프로젝트에 병합해줘
```

`new` 모드(기본)는 새 프로젝트로 생성하고, `merge` 모드는 기존 프로젝트에 이력을 추가합니다.

### 고급 히스토리 조회

```
"웹 개발" 프로젝트에서 70점 이상 react 태그 항목만 보여줘
이번 달에 분석한 D등급 프롬프트를 찾아줘
출력 형식이 누락된 항목을 모두 보여줘
```

`query_history`가 지원하는 필터 조건:

| 조건 | 예시 |
|------|------|
| 점수 범위 | `minScore: 70`, `maxScore: 90` |
| 등급 | `grade: A` / `B` / `C` / `D` |
| 날짜 | `dateFrom: 2026-03-01`, `dateTo: 2026-03-31` |
| 태그 | `tags: ["react", "api"]` |
| 키워드 | `keyword: "로그인"` |
| 정렬 | `sortBy: score`, `order: desc` |

### 스냅샷 비교

스프린트 시작/종료 시점을 스냅샷으로 저장하면 기간별 개선을 수치로 확인할 수 있습니다.

```
"웹 개발" 프로젝트 스냅샷을 "sprint-4-start"로 저장해줘

(2주 후)
스프린트 전후 변화를 비교해줘
```

→ 엔트리 수, 평균 점수, 등급 분포, 신규 태그 변화를 한눈에 보여줍니다.

## Step 5. 프롬프트 버전 관리

프롬프트를 개선하면서 이전 버전과 비교할 수 있습니다.

**프롬프트 개선 후 버전 연결:**

첫 번째 분석 후 받은 `entryId`를 기반으로 개선된 프롬프트를 다시 분석하면 자동으로 버전 체인이 만들어집니다:

```
"REST API 설계해줘"를 분석하고 "웹 개발" 프로젝트에 저장해줘
```

→ entryId: `h_1234` 를 받음. 개선된 프롬프트를 이전 버전과 연결:

```
"너는 API 아키텍트야. 이커머스 플랫폼용 REST API를 설계해줘..."를 분석하고
"웹 개발" 프로젝트에 저장해줘. 이전 버전은 h_1234야.
```

**두 프롬프트 비교 (diff):**

```
h_1234와 h_5678을 비교해줘 >> diff
```

텍스트 diff, 점수 변화, 5축 변화, 태그 변화를 한눈에 보여줍니다:

```
D(25) → A(91) [+66점]

텍스트 diff:
  [-REST API 설계해줘-]
  [+너는 API 아키텍트야. 이커머스 플랫폼용 REST API를 설계해줘...+]

5축 변화:
  clarity:       30 → 90 (+60)
  specificity:   15 → 92 (+77)
  context:       20 → 88 (+68)
```

**버전 이력 조회:**

```
h_1234의 버전 이력을 보여줘
```

→ v1(25점, D) → v2(91점, A) [+66점 개선]

## Step 6. 대시보드 시각화

프로젝트의 프롬프트 히스토리를 차트로 시각화할 수 있습니다. HTML 대시보드가 생성되어 브라우저에서 확인 가능합니다.

**프로젝트별 대시보드:**

```
"웹 개발" 프로젝트를 시각화해줘 >> viz
```

점수 추이(라인), 5축 레이더(최근 vs 평균), 등급 분포(도넛), 태그 통계(바), 최근 분석 기록 테이블을 포함합니다.

**전체 프로젝트 대시보드:**

```
PromptLens 전체 대시보드를 보여줘 >> 시각화
```

생성된 파일은 `~/.promptlens/dashboard-*.html`에 저장됩니다.

## Step 7. API 모드 활성화

더 정밀한 분석이 필요하면 Anthropic API 키를 등록하여 API 모드를 사용할 수 있습니다.

### API 키 발급

1. [Anthropic Console](https://console.anthropic.com/)에 로그인
2. 좌측 메뉴에서 **API Keys** 선택
3. **Create Key** 클릭 → 이름 입력 → 생성
4. 생성된 키(`sk-ant-api03-...`)를 복사

### API 키 등록

Claude 대화에서:

```
PromptLens API 키를 설정해줘: sk-ant-api03-xxxxx
```

키가 자동으로 검증되고 로컬(`~/.promptlens/settings.json`)에 저장됩니다.

### API 모드로 분석

```
이 프롬프트를 API 모드로 분석해줘: "React로 로그인 폼 만들어줘"
```

API 모드는 3색 리포트를 제공합니다:

| 색상 | 의미 | 설명 |
|------|------|------|
| ✅ **Referenced** | 명시적 언급 | 프롬프트에 직접 적힌 정보 |
| 🟡 **Inferred** | AI 추론 | 명시하지 않았지만 AI가 추측하는 정보 (+ 신뢰도) |
| ❌ **Missing** | 누락 | 프롬프트에 없고 추론도 어려운 정보 |

> API 모드는 사용자의 API 키로 Claude API를 호출하므로 별도 비용이 발생합니다. 일상적 분석에는 무료인 local 모드를 권장합니다.
