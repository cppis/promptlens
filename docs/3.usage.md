## 1. 설치

Prompatic는 npm 패키지(`prompatic`)로 배포됩니다. **npx로 설치 없이 바로 실행**할 수 있습니다.

### 1.1 Claude Desktop에 등록

`claude_desktop_config.json`에 다음을 추가합니다.

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "prompatic": {
      "command": "npx",
      "args": ["-y", "@cppis/promptic"]
    }
  }
}
```

설정 후 Claude Desktop을 재시작하면 Prompatic 도구가 활성화됩니다. 별도 설치가 필요 없으며, npx가 자동으로 최신 버전을 실행합니다.

### 1.2 Claude Code에 등록

```bash
claude mcp add prompatic -- npx -y prompatic
```

등록 후 바로 사용 가능합니다.

### 1.3 글로벌 설치 (선택)

npx 대신 글로벌로 설치하면 실행 속도가 더 빠릅니다.

```bash
npm install -g prompatic
```

Claude Desktop 설정:

```json
{
  "mcpServers": {
    "prompatic": {
      "command": "prompatic"
    }
  }
}
```

Claude Code:

```bash
claude mcp add prompatic prompatic
```

### 1.4 소스에서 직접 실행 (개발용)

소스를 수정하거나 디버깅하려면:

```bash
git clone https://github.com/cppis/prompt-lens.git
cd prompt-lens/mcp-server
npm install
```

Claude Desktop 설정:

```json
{
  "mcpServers": {
    "prompatic": {
      "command": "node",
      "args": ["/absolute/path/to/prompt-lens/mcp-server/index.js"]
    }
  }
}
```

## 2. 초기 설정

MCP 서버를 등록하면 **local 모드는 즉시 사용 가능**합니다. API 키 없이 프롬프트 분석, 프로젝트 관리, 히스토리 기능을 모두 쓸 수 있습니다.

### 2.1 분석 모드 선택

| 모드 | API 키 | 비용 | 분석 방식 | 추천 용도 |
|------|--------|------|-----------|-----------|
| **local** (기본) | 불필요 | 무료 | 규칙 엔진 5축 점수 + 누락 요소 + 개선 제안 | 일상적 프롬프트 점검 |
| **api** (선택) | 필요 (BYOK) | 유료 | Claude API 3색 리포트 (Referenced/Inferred/Missing) | 중요 프롬프트 정밀 분석 |

### 2.2 API Key 설정 (api 모드 사용 시)

Claude Desktop/Code에서 자연어로 설정할 수 있습니다:

```
"Prompatic API 키를 설정해줘: sk-ant-api03-..."
```

`set_api_key` 도구가 호출되어 키 유효성 검증 후 `~/.prompatic/settings.json`에 저장됩니다.

#### API Key 발급

1. [Anthropic Console](https://console.anthropic.com/) 접속 → 로그인
2. 좌측 메뉴에서 **API Keys** 선택
3. **Create Key** 클릭 → 이름 입력 (예: `Prompatic`) → 생성
4. 생성된 키(`sk-ant-api03-...`)를 복사

### 2.3 모델 선택 가이드

| 모델 | 속도 | 품질 | 비용 (입력/출력 per 1M) | 추천 |
|------|------|------|-------------------------|------|
| Claude Sonnet 4.6 (기본) | 빠름 | 높음 | $3 / $15 | 일반 정밀 분석 |
| Claude Haiku 4.5 | 매우 빠름 | 보통 | $0.80 / $4 | 빠른 반복 분석 |
| Claude Opus 4.6 | 느림 | 최고 | $15 / $75 | 최고 품질 분석 |

### 2.4 비용 참고

api 모드는 사용자의 Anthropic API 키로 Claude API를 호출합니다. Claude Desktop/Code 구독과 별도로 API 사용료가 발생합니다. 프롬프트 1건 분석 시 Sonnet 4.6 기준 약 $0.003~$0.05입니다. 일상적 분석에는 무료인 local 모드를 권장합니다.

## 3. MCP 도구별 사용법

Prompatic는 17개의 MCP 도구와 3개의 MCP Prompts 워크플로우를 제공합니다. Claude Desktop/Code에서 자연어 또는 커맨드 패턴(`>> anz`, `>> deep` 등)으로 요청하면 적절한 도구가 자동 호출됩니다.

### 3.1 analyze_prompt — 프롬프트 분석

프롬프트의 품질을 분석하여 점수, 누락 요소, 개선 제안을 반환합니다.

**파라미터:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `prompt` | ✅ | 분석할 프롬프트 텍스트 |
| `mode` | - | `"local"` (기본) 또는 `"api"` |
| `projectId` | - | 분석 결과를 저장할 프로젝트 ID. 미지정 시 활성 프로젝트(`set_active_project`)로 자동 폴백 |
| `tags` | - | 히스토리에 부여할 태그 배열 |
| `parentId` | - | 이전 버전 entryId (버전 체인 연결, 자동 diff) |
| `autoRun` | - | `true`면 개선된 프롬프트를 즉시 실행 (`>> anz+run` 트리거) |

**커맨드 패턴:**

| 커맨드 | 동작 |
|--------|------|
| `... >> anz` / `... >> 분석` | 빠른 분석 |
| `... >> anz+run` / `... >> 분석+실행` | 분석 + 개선 프롬프트 즉시 실행 |
| `... >> deep` / `... >> 정밀분석` | 4단계 정밀 분석 파이프라인 |

**자연어 트리거 (자동 감지):**

영어: "analyze this prompt", "check this prompt", "rate this prompt", "evaluate my prompt", "review this prompt"

한글: "이 프롬프트 분석해줘", "프롬프트 점검해줘", "프롬프트 평가해줘", "이 프롬프트 어때", "프롬프트 좀 봐줘"

암시적: "이거 괜찮아?", "이거 좀 부족한데", "how can I improve this"

**자연어 사용 예시:**

```
"이 프롬프트를 분석해줘: You are a backend engineer. Design a REST API for..."
```

```
"이 프롬프트를 API 모드로 정밀 분석해줘: ..."
```

**local 모드 응답 예시:**

```json
{
  "mode": "local",
  "score": 62,
  "grade": "C",
  "axisScores": {
    "clarity": 70, "specificity": 55, "context": 60,
    "structure": 65, "actionability": 58
  },
  "missingElements": ["output_format", "examples", "constraints"],
  "suggestions": ["출력 형식을 명시하세요", "구체적인 예시를 추가하세요"],
  "enhancedPrompt": "..."
}
```

**5축(Axis) 점수 기준:**

| 축 | 측정 기준 |
|------|-----------|
| Clarity | 지시가 명확하고 모호하지 않은가 |
| Specificity | 구체적 수치·이름·조건이 포함되었는가 |
| Context | 배경 정보가 충분한가 |
| Structure | 논리적 구조(단계, 형식)가 있는가 |
| Actionability | AI가 바로 실행할 수 있는 수준인가 |

**등급 기준:** A(90+), B(70~89), C(50~69), D(50 미만)

### 3.2 list_projects / create_project — 프로젝트 관리

프롬프트를 주제별로 묶어 관리할 수 있습니다.

**create_project:** `name` (필수) — 프로젝트 이름

**list_projects:** 파라미터 없음. 모든 프로젝트를 통계와 함께 반환.

```
"'Backend API Design' 프로젝트를 만들어줘"
"프로젝트 목록을 보여줘"
```

### 3.3 get_history / add_history_entry — 히스토리 관리

**get_history:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `projectId` | ✅ | 프로젝트 ID |
| `limit` | - | 반환 최대 개수 (기본: 20) |
| `search` | - | 프롬프트·태그·노트 검색어 |

**add_history_entry:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `projectId` | ✅ | 프로젝트 ID |
| `prompt` | ✅ | 프롬프트 텍스트 |
| `score` | - | 수동 점수 (0~100, 미지정 시 자동 채점) |
| `tags` | - | 태그 배열 |
| `note` | - | 메모 |

### 3.4 import_claude_conversations — Claude 대화 Import

Claude Desktop과 Claude Code의 대화 히스토리를 Prompatic로 가져옵니다.

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `filePath` | ✅ | `conversations.json` 또는 `.jsonl` 파일 경로 |
| `mode` | - | `"per-conversation"` (기본) 또는 `"single"` |
| `projectName` | - | single 모드 프로젝트 이름 |

**지원 소스:**

| 소스 | 파일 형식 | 획득 방법 |
|------|-----------|-----------|
| Claude Desktop | `.json` | Settings > Privacy > Export Data |
| Claude Code | `.jsonl` | `~/.claude/projects/<project>/` |

```
"~/Downloads/conversations.json 파일을 Import해줘"
```

**Import 규칙:** 10자 미만 스킵, 중복 제거 (앞 200자), 소스 태그 자동 부여, AI 응답 앞 300자가 노트에 저장.

### 3.5 get_stats — 통계 조회

파라미터 없음. 총 프로젝트 수, 총 프롬프트 수, 평균 점수, 태그 Top 10을 반환합니다.

```
"전체 통계를 보여줘"
```

### 3.6 set_api_key / get_settings — 설정 관리

**set_api_key:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `apiKey` | ✅ | Anthropic API 키 (`sk-ant-...`) |
| `model` | - | API 분석 모델 (기본: `claude-sonnet-4-6`) |

키 등록 시 실제 API 호출로 유효성을 검증합니다.

**get_settings:** 파라미터 없음. API 키 상태, 모델, 활성 프로젝트, 저장 경로, 사용 가능한 분석 모드를 반환합니다.

### 3.6-B set_active_project — 활성 프로젝트 설정

프로젝트를 한 번 활성으로 지정하면, 이후 `analyze_prompt` 호출 시 `projectId`를 생략해도 자동으로 저장되고 `entryId`가 항상 발급됩니다.

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `action` | ✅ | `"set"` (설정) / `"clear"` (해제) / `"get"` (확인) |
| `projectName` | `set`일 때 ✅ | 활성으로 지정할 프로젝트 이름 |

**동작 흐름:**
- `set` — 프로젝트 이름으로 ID를 조회한 뒤 `settings.json`에 저장. 존재하지 않는 이름이면 사용 가능한 프로젝트 목록을 반환.
- `clear` — `settings.json`에서 `activeProjectId` 키를 제거.
- `get` — 현재 활성 프로젝트 ID와 이름을 반환.

```
"웹 개발"을 활성 프로젝트로 설정해줘   → action: set, projectName: "웹 개발"
지금 활성 프로젝트가 뭐야?              → action: get
활성 프로젝트 해제해줘                  → action: clear
```

> 활성 프로젝트 설정은 `~/.prompatic/settings.json`에 저장되며 Claude Desktop 재시작 후에도 유지됩니다.

### 3.7 visualize_project — 대시보드 시각화

프로젝트의 분석 히스토리를 Chart.js 기반 HTML 대시보드로 생성합니다.

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `projectId` | - | 프로젝트 ID (미지정 시 전체 프로젝트 오버뷰) |
| `outputPath` | - | 저장 경로 (기본: `~/.prompatic/dashboard-*.html`) |

```
"웹 개발" 프로젝트를 시각화해줘 >> viz
```

차트: 점수 추이(라인), 5축 레이더(최근 vs 평균), 등급 분포(도넛), 태그 통계(바), 최근 분석 테이블.

### 3.8 compare_prompts — 프롬프트 비교 (diff)

두 프롬프트 엔트리 간 텍스트 diff, 점수 변화, 5축 변화, 태그 변화를 비교합니다.

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `entryIdA` | ✅ | 비교 대상 A (이전 버전) |
| `entryIdB` | ✅ | 비교 대상 B (이후 버전) |
| `projectId` | - | 프로젝트 ID (검색 범위 제한) |

```
h_1234와 h_5678을 비교해줘 >> diff
```

Word-level diff(LCS 기반)로 `[-삭제됨-]` / `[+추가됨+]` 형태로 변경점을 보여줍니다.

### 3.9 get_versions — 버전 체인 조회

특정 프롬프트 엔트리의 전체 버전 이력(parentId 체인)을 반환합니다.

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `entryId` | ✅ | 조회할 엔트리 ID |
| `projectId` | ✅ | 프로젝트 ID |

```
h_1234의 버전 이력을 보여줘
```

→ v1(25점, D) → v2(55점, C) → v3(91점, A) [+66점 개선]

### 3.10 MCP Prompts 워크플로우

Claude Desktop의 `/` 메뉴에서 선택할 수 있는 워크플로우 프리셋입니다.

| 프리셋 | 설명 | 인자 |
|--------|------|------|
| `quick-analyze` | 빠른 5축 분석 + 개선 제안 | `prompt` (필수) |
| `deep-analyze` | 4단계 정밀 파이프라인 (분석→개선→재분석→비교) | `prompt` (필수), `project` (선택) |
| `project-report` | 프로젝트 대시보드 생성 + 통계 요약 | `project` (필수) |

### 3.11 export_project — 로컬 파일 내보내기

프로젝트의 히스토리를 로컬 파일로 내보냅니다. 백업, 팀 공유, 스프레드시트 분석에 활용합니다.

**파라미터:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `projectId` | ✅ | 내보낼 프로젝트 ID |
| `format` | - | `"json"` (기본), `"markdown"`, `"csv"` |
| `outputPath` | - | 저장 경로 (기본: `~/.prompatic/exports/`) |
| `scoreMin` / `scoreMax` | - | 점수 범위 필터 (0~100) |
| `grade` | - | 등급 필터 (예: `"A"`, `"B"`) |
| `dateFrom` / `dateTo` | - | 날짜 범위 필터 (ISO 8601) |
| `tags` | - | AND 조건 태그 필터 배열 |

**형식별 특징:**

| 형식 | 용도 | 재가져오기 |
|------|------|-----------|
| `json` | 완전한 데이터 보존, 팀 공유 | ✅ (import_project로) |
| `markdown` | 사람이 읽기 좋은 리포트 | ❌ |
| `csv` | 스프레드시트(Excel/Sheets) 분석 | ❌ |

```
"웹 개발" 프로젝트를 JSON으로 내보내줘
"Backend" 프로젝트에서 A등급 항목만 Markdown으로 내보내줘
지난달 분석만 CSV로 내보내줘 (dateFrom: 2026-02-01, dateTo: 2026-02-28)
```

### 3.12 import_project — 로컬 파일 가져오기

`export_project`로 내보낸 `.prompatic.json` 파일을 다시 가져옵니다. 다른 컴퓨터에서 이어서 작업하거나 팀원과 데이터를 공유할 때 사용합니다.

**파라미터:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `filePath` | ✅ | 가져올 `.prompatic.json` 파일 경로 |
| `mode` | - | `"new"` (기본, 새 프로젝트로 생성) / `"merge"` (기존 프로젝트에 병합) |
| `projectName` | - | `new` 모드 시 새 프로젝트 이름 (미지정 시 원본 이름 유지) |

```
~/Downloads/backend-api.prompatic.json을 가져와줘
이 파일을 "공유된 프로젝트"라는 이름으로 새로 만들어줘
```

### 3.13 query_history — 고급 히스토리 조회

다양한 조건을 조합하여 히스토리를 필터링하고 정렬합니다. `get_history`보다 강력한 쿼리를 지원합니다.

**파라미터:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `projectId` | ✅ | 조회할 프로젝트 ID |
| `scoreMin` / `scoreMax` | - | 점수 범위 (0~100) |
| `grade` | - | 등급 필터 배열 (예: `["A", "B"]`) |
| `dateFrom` / `dateTo` | - | 날짜 범위 (ISO 8601) |
| `tags` | - | AND 조건 태그 배열 |
| `tagsAny` | - | OR 조건 태그 배열 |
| `missing` | - | 누락 요소 필터 (예: `["output_format"]`) |
| `versionsOnly` | - | `true`면 버전 체인 항목만 반환 |
| `search` | - | 프롬프트·노트·태그 전문 검색 |
| `limit` | - | 최대 반환 개수 (기본: 20) |
| `sortBy` | - | `"score_desc"` (기본), `"score_asc"`, `"date_desc"`, `"date_asc"` |

**응답 예시:**

```json
{
  "entries": [...],
  "total": 12,
  "filter": "점수 70~100, 태그 react, 최근 순",
  "projectName": "웹 개발"
}
```

```
"웹 개발" 프로젝트에서 70점 이상 react 태그만 보여줘
이번 달에 분석한 D등급 프롬프트를 찾아줘
출력 형식이 누락된 항목을 모두 보여줘
```

### 3.14 snapshot_project — 스냅샷 & 비교

특정 시점의 프로젝트 상태를 파일로 저장하고, 두 스냅샷 간 변화를 비교합니다. 스프린트 단위 회고나 개선 추적에 활용합니다.

**파라미터:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `projectId` | ✅ | 스냅샷을 찍을 프로젝트 ID |
| `label` | - | 스냅샷 이름 (예: `"sprint-3-end"`, `"before-review"`) |
| `compareWith` | - | 비교할 이전 스냅샷 파일 경로 |

**스냅샷 저장 경로:** `~/.prompatic/snapshots/{projectId}-{timestamp}.json`

**비교 결과 예시:**

```
스냅샷 비교 (before-refactor → sprint-4-end)
  엔트리 수:  12 → 18 (+6)
  평균 점수:  58 → 74 (+16)
  A등급:       1 → 5  (+4)
  D등급:       6 → 1  (−5)
  새 태그:     typescript, error-handling
```

```
"웹 개발" 프로젝트 스냅샷을 "sprint-4-start"로 저장해줘
이번 스프린트 전후 변화를 비교해줘
  (compareWith: ~/.prompatic/snapshots/proj_xxx-sprint-3-end.json)
```

## 6. FAQ

**Q: MCP 서버를 등록했는데 도구가 보이지 않아요.**

Claude Desktop을 완전히 종료하고 재시작하세요. `claude_desktop_config.json`의 JSON 형식이 올바른지 확인하세요. Claude Code는 `claude mcp list`로 확인합니다.

**Q: npx 실행이 느린데 더 빠르게 할 수 있나요?**

`npm install -g prompatic`로 글로벌 설치하면 매번 다운로드하지 않아 더 빠릅니다. Claude Desktop 설정에서 `"command": "prompatic"`로 변경하세요.

**Q: API 키 없이도 사용할 수 있나요?**

네. 기본 분석 모드(local)는 API 키 없이 즉시 사용 가능합니다. API 키는 정밀 분석(api 모드)에서만 필요합니다.

**Q: API 키는 어디에 저장되나요?**

`~/.prompatic/settings.json`에 로컬로 저장됩니다. 외부 서버로 전송되지 않습니다.

**Q: api 모드의 비용은?**

프롬프트 1건당 Sonnet 4.6 기준 약 $0.003~$0.05입니다. Claude Desktop/Code 구독과 별도입니다.

**Q: 데이터를 백업하려면?**

`~/.prompatic/` 디렉토리 전체를 복사하거나, `export_project` 도구로 프로젝트별 `.prompatic.json` 파일을 생성하세요. JSON 형식은 `import_project`로 완전히 복원 가능합니다.

**Q: 여러 컴퓨터에서 동기화할 수 있나요?**

`export_project` → 파일 공유 → `import_project` 흐름으로 수동 동기화가 가능합니다. 자동화하려면 `~/.prompatic/`를 Dropbox, iCloud 등 클라우드 동기화 폴더에 심볼릭 링크하세요.

**Q: v0.3.0 data.json 데이터는 어떻게 되나요?**

v0.4.0 서버 첫 실행 시 자동으로 프로젝트별 파일(`~/.prompatic/projects/`)로 마이그레이션됩니다. 원본 `data.json`은 `data.json.migrated`로 보존되므로 데이터 손실은 없습니다.

**Q: 특정 조건으로 히스토리를 검색하려면?**

`query_history` 도구를 사용하세요. 점수 범위, 등급, 날짜, 태그(AND/OR), 누락 요소, 키워드를 조합한 고급 필터와 정렬을 지원합니다.
