## 1. 설치

PromptLens는 MCP(Model Context Protocol) Server로 동작합니다. Claude Desktop 또는 Claude Code에 MCP 서버를 등록하면 9개의 도구를 자연어로 사용할 수 있습니다.

### 1.1 MCP Server 설치

```bash
cd mcp-server
npm install
```

### 1.2 Claude Desktop에 등록

`claude_desktop_config.json`에 다음을 추가합니다.

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "promptlens": {
      "command": "node",
      "args": ["/absolute/path/to/mcp-server/index.js"]
    }
  }
}
```

설정 후 Claude Desktop을 재시작하면 PromptLens 도구가 활성화됩니다.

### 1.3 Claude Code에 등록

```bash
claude mcp add promptlens node /absolute/path/to/mcp-server/index.js
```

등록 후 바로 사용 가능합니다.

## 2. 초기 설정

MCP 서버를 등록하면 **local 모드는 즉시 사용 가능**합니다. API 키 없이 프롬프트 분석, 프로젝트 관리, 히스토리 기능을 모두 쓸 수 있습니다.

### 2.1 분석 모드 선택

| 모드 | API 키 | 비용 | 분석 방식 | 추천 용도 |
|------|--------|------|-----------|-----------|
| **local** (기본) | 불필요 | 무료 | 규칙 엔진 5축 점수 + 누락 요소 + 개선 제안 | 일상적 프롬프트 점검 |
| **api** (선택) | 필요 (BYOK) | 유료 | Claude API 3색 리포트 (Referenced/Inferred/Missing) | 중요 프롬프트 정밀 분석 |

### 2.2 API Key 설정 (api 모드 사용 시)

Claude Desktop/Code에서 자연어로 설정할 수 있습니다:

```
"PromptLens API 키를 설정해줘: sk-ant-api03-..."
```

`set_api_key` 도구가 호출되어 키 유효성 검증 후 `~/.promptlens/settings.json`에 저장됩니다.

#### API Key 발급

1. [Anthropic Console](https://console.anthropic.com/) 접속 → 로그인
2. 좌측 메뉴에서 **API Keys** 선택
3. **Create Key** 클릭 → 이름 입력 (예: `PromptLens`) → 생성
4. 생성된 키(`sk-ant-api03-...`)를 복사

### 2.3 모델 선택 가이드

| 모델 | 속도 | 품질 | 비용 (입력/출력 per 1M) | 추천 |
|------|------|------|-------------------------|------|
| Claude Sonnet 4.5 (기본) | 빠름 | 높음 | $3 / $15 | 일반 정밀 분석 |
| Claude Haiku 3.5 | 매우 빠름 | 보통 | $0.80 / $4 | 빠른 반복 분석 |
| Claude Opus 4 | 느림 | 최고 | $15 / $75 | 최고 품질 분석 |

### 2.4 비용 참고

api 모드는 사용자의 Anthropic API 키로 Claude API를 호출합니다. Claude Desktop/Code 구독과 별도로 API 사용료가 발생합니다. 프롬프트 1건 분석 시 Sonnet 4.5 기준 약 $0.003~$0.05입니다. 일상적 분석에는 무료인 local 모드를 권장합니다.

## 3. MCP 도구별 사용법

PromptLens는 9개의 MCP 도구를 제공합니다. Claude Desktop/Code에서 자연어로 요청하면 적절한 도구가 자동 호출됩니다.

### 3.1 analyze_prompt — 프롬프트 분석

프롬프트의 품질을 분석하여 점수, 누락 요소, 개선 제안을 반환합니다.

**파라미터:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `prompt` | ✅ | 분석할 프롬프트 텍스트 |
| `mode` | - | `"local"` (기본) 또는 `"api"` |
| `projectId` | - | 분석 결과를 저장할 프로젝트 ID |
| `tags` | - | 히스토리에 부여할 태그 배열 |

**자연어 사용 예시:**

```
"이 프롬프트를 분석해줘: You are a backend engineer. Design a REST API for..."
```

```
"이 프롬프트를 API 모드로 정밀 분석해줘: ..."
```

**local 모드 응답 예시:**

```json
{
  "mode": "local",
  "score": 62,
  "grade": "C",
  "axisScores": {
    "clarity": 70, "specificity": 55, "context": 60,
    "structure": 65, "actionability": 58
  },
  "missingElements": ["output_format", "examples", "constraints"],
  "suggestions": ["출력 형식을 명시하세요", "구체적인 예시를 추가하세요"],
  "enhancedPrompt": "..."
}
```

**5축(Axis) 점수 기준:**

| 축 | 측정 기준 |
|------|-----------|
| Clarity | 지시가 명확하고 모호하지 않은가 |
| Specificity | 구체적 수치·이름·조건이 포함되었는가 |
| Context | 배경 정보가 충분한가 |
| Structure | 논리적 구조(단계, 형식)가 있는가 |
| Actionability | AI가 바로 실행할 수 있는 수준인가 |

**등급 기준:** A(90+), B(70~89), C(50~69), D(50 미만)

### 3.2 list_projects / create_project — 프로젝트 관리

프롬프트를 주제별로 묶어 관리할 수 있습니다.

**create_project:** `name` (필수) — 프로젝트 이름

**list_projects:** 파라미터 없음. 모든 프로젝트를 통계와 함께 반환.

```
"PromptLens에 'Backend API Design' 프로젝트를 만들어줘"
"PromptLens 프로젝트 목록을 보여줘"
```

### 3.3 get_history / add_history_entry — 히스토리 관리

**get_history:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `projectId` | ✅ | 프로젝트 ID |
| `limit` | - | 반환 최대 개수 (기본: 20) |
| `search` | - | 프롬프트·태그·노트 검색어 |

**add_history_entry:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `projectId` | ✅ | 프로젝트 ID |
| `prompt` | ✅ | 프롬프트 텍스트 |
| `score` | - | 수동 점수 (0~100, 미지정 시 자동 채점) |
| `tags` | - | 태그 배열 |
| `note` | - | 메모 |

### 3.4 import_claude_conversations — Claude 대화 Import

Claude Desktop과 Claude Code의 대화 히스토리를 PromptLens로 가져옵니다.

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `filePath` | ✅ | `conversations.json` 또는 `.jsonl` 파일 경로 |
| `mode` | - | `"per-conversation"` (기본) 또는 `"single"` |
| `projectName` | - | single 모드 프로젝트 이름 |

**지원 소스:**

| 소스 | 파일 형식 | 획득 방법 |
|------|-----------|-----------|
| Claude Desktop | `.json` | Settings > Privacy > Export Data |
| Claude Code | `.jsonl` | `~/.claude/projects/<project>/` |

```
"~/Downloads/conversations.json 파일을 PromptLens에 Import해줘"
```

**Import 규칙:** 10자 미만 스킵, 중복 제거 (앞 200자), 소스 태그 자동 부여, AI 응답 앞 300자가 노트에 저장.

### 3.5 get_stats — 통계 조회

파라미터 없음. 총 프로젝트 수, 총 프롬프트 수, 평균 점수, 태그 Top 10을 반환합니다.

```
"PromptLens 전체 통계를 보여줘"
```

### 3.6 set_api_key / get_settings — 설정 관리

**set_api_key:**

| 파라미터 | 필수 | 설명 |
|----------|------|------|
| `apiKey` | ✅ | Anthropic API 키 (`sk-ant-...`) |
| `model` | - | API 분석 모델 (기본: `claude-sonnet-4-5-20250514`) |

키 등록 시 실제 API 호출로 유효성을 검증합니다.

**get_settings:** 파라미터 없음. API 키 상태, 모델, 저장 경로, 사용 가능한 분석 모드를 반환합니다.

## 4. 사용 시나리오

### 시나리오 1: 프롬프트 품질 빠른 점검 (local 모드)

```
나: 이 프롬프트를 분석해줘: "Design a microservices architecture"

Claude: [analyze_prompt 호출]
  - 종합 점수: 25점 (D등급)
  - 약점: Role(0), Context(0), Format(0), Example(0)
  - 제안: 역할 정의, 배경 맥락, 출력 형식, 예시를 추가하세요.

나: 개선된 프롬프트를 다시 분석해줘

Claude: [analyze_prompt 호출]
  - 종합 점수: 88점 (B등급) — 63점 상승!
```

### 시나리오 2: 프로젝트별 프롬프트 관리

```
나: PromptLens에 'Code Review Assistant' 프로젝트를 만들어줘

나: 이 프롬프트를 분석하고 'Code Review Assistant'에 저장해줘: "Review this Go code..."
    태그는 go, code-review로 해줘

나: (1주일 후) 'Code Review Assistant' 프로젝트의 분석 이력을 보여줘
```

### 시나리오 3: Claude 대화 히스토리 분석

```
나: ~/Downloads/conversations.json을 PromptLens에 Import해줘

Claude: 32개 대화에서 187개 프롬프트 Import 완료 (15개 스킵)

나: Import된 프로젝트 목록을 보여줘
```

### 시나리오 4: 중요 프롬프트 정밀 분석 (api 모드)

```
나: PromptLens API 키를 설정해줘: sk-ant-api03-xxxxx

나: 이 프롬프트를 API 모드로 분석해줘: "You are an expert financial advisor..."

Claude: [analyze_prompt mode:api]
  ✅ Referenced: 역할 정의, 전문성 수준, 대상 고객
  🟡 Inferred: 응답 언어를 영어로 추론, 투자 유형을 주식으로 가정
  ❌ Missing: 리스크 수준 미지정, 투자 기간 없음, 출력 형식 미지정
```

## 5. 개발자 가이드

### 5.1 프로젝트 구조

```
mcp-server/
├── index.js                    ← MCP Server 엔트리 (9개 도구)
├── lib/
│   ├── analyzer.js             ← 분석 엔진 (local + api)
│   ├── storage.js              ← 파일 기반 저장소
│   └── importer.js             ← Claude 대화 Import
├── test/
│   ├── storage.test.js         ← 19개 테스트
│   ├── analyzer.test.js        ← 25개 테스트
│   ├── importer.test.js        ← 16개 테스트
│   └── mcp-tools.test.js       ← 8개 통합 테스트
└── package.json
```

### 5.2 테스트 실행

```bash
cd mcp-server

# 전체 (68개)
npm test

# 개별
node --test test/storage.test.js     # 19 tests
node --test test/analyzer.test.js    # 25 tests
node --test test/importer.test.js    # 16 tests
node --test test/mcp-tools.test.js   # 8 tests (~40초)
```

### 5.3 데이터 저장 경로

| 파일 | 경로 | 내용 |
|------|------|------|
| data.json | `~/.promptlens/data.json` | 프로젝트, 히스토리 엔트리 |
| settings.json | `~/.promptlens/settings.json` | API 키, 모델 |

### 5.4 분석 엔진 커스터마이징

`lib/analyzer.js`의 `analyzePrompt()` 함수에서 5축 점수 계산 로직, 누락 요소 판별 조건, 개선 제안 문구를 커스터마이징할 수 있습니다. 한국어 패턴(전문가, 현재, 단계, 예를 들, 제한)도 지원합니다.

## 6. FAQ

**Q: MCP 서버를 등록했는데 도구가 보이지 않아요.**

Claude Desktop을 완전히 종료하고 재시작하세요. `claude_desktop_config.json`의 경로가 절대 경로인지 확인하세요. Claude Code는 `claude mcp list`로 확인합니다.

**Q: API 키 없이도 사용할 수 있나요?**

네. 기본 분석 모드(local)는 API 키 없이 즉시 사용 가능합니다. API 키는 정밀 분석(api 모드)에서만 필요합니다.

**Q: API 키는 어디에 저장되나요?**

`~/.promptlens/settings.json`에 로컬로 저장됩니다. 외부 서버로 전송되지 않습니다.

**Q: api 모드의 비용은?**

프롬프트 1건당 Sonnet 4.5 기준 약 $0.003~$0.05입니다. Claude Desktop/Code 구독과 별도입니다.

**Q: 데이터를 백업하려면?**

`~/.promptlens/` 디렉토리를 복사하면 됩니다.

**Q: 여러 컴퓨터에서 동기화할 수 있나요?**

현재는 로컬 파일 저장만 지원합니다. `~/.promptlens/`를 클라우드 동기화 서비스로 심볼릭 링크하면 수동 동기화가 가능합니다.
