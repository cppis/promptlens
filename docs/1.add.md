## 1. 제품 설계

### 1.1 컨셉

> "AI가 내 말을 어떻게 이해했는지 보여주고, 부족한 맥락을 알아서 채워준다."

**제품 방향:** "교육"이 아니라 "즉시 실용" — AI 답변이 왜 이상한지 바로 보여주고, 다음에 뭘 보충하면 되는지 안내하며, 반복되는 약점은 자동으로 코칭해주는 MCP 도구.

**3가지 핵심 축:**

| 축 | 기능 | 사용자 가치 |
|---|------|----------|
| AI 해석 투명화 | 프롬프트를 넣으면 AI가 뭘 이해했고 뭘 모르는지 보여줌 | "AI 답변이 왜 이상한지" 즉시 파악 |
| AI 해석 시뮬레이션 | AI가 1인칭으로 "나는 이렇게 이해했고 여기서 추측했다"를 서술 | 점수가 아니라 AI의 사고 과정을 직접 체험 |
| 누적 패턴 코칭 | 분석 이력에서 반복 약점 추출 → 개인화 코칭 + 행동 제안 | 일회성 분석 → 지속적 성장 도구 |

**보조 기능 (확장 로드맵):**

| 기능 | 설명 |
|------|------|
| 맥락 빌더 | 프로젝트·역할·선호 스타일을 프로필로 저장, 필요한 맥락만 선별·삽입 |
| 프롬프트 변이 비교 (Prompt Lab) | 같은 의도의 프롬프트 2~3개를 변형 생성, AI 해석 차이를 나란히 비교 |
| 응답 품질 감사 (Response Auditor) | AI 답변의 환각, 논리 비약, 출처 불명확 주장 자동 탐지 |

**기술 스택:**

| 계층 | 기술 | 상태 |
|------|------|------|
| 클라이언트 | MCP Server (Node.js + @modelcontextprotocol/sdk, v0.3.0) | 확정 (현재) |
| AI 연동 | Claude API 직접 호출 (BYOK, api 모드) | 확정 |
| 분석 방식 | local 모드 (로컬 규칙, 무료), api 모드 (Claude API, BYOK) | 확정 |
| 저장소 | JSON 파일 (~/.promptlens/data.json, settings.json) | 확정 |
| 비용 모델 | 사용자 본인 API 키 사용 (api 모드만) | 확정 |

### 1.2 핵심 철학 — AI 활용 역량 5가지

사용자가 AI를 더 잘 쓰려면 5가지 역량이 필요하고, PromptLens는 도구 사용 과정에서 이를 자연스럽게 키워준다.

| 역량 | 정의 | PromptLens가 돕는 방식 |
|------|------|---------------------|
| AI 해석 이해 | AI가 내 말을 어떻게 읽는지 아는 것 | 해석 투명화 + 해석 시뮬레이션 |
| 맥락 관리 | AI에게 필요한 배경 정보를 효율적으로 전달 | 맥락 빌더 (향후) |
| 결과 검증 | AI 답변의 품질과 신뢰도를 판단 | Response Auditor (향후) |
| 실험적 접근 | 다양한 표현으로 시도하고 비교하는 습관 | Prompt Lab (향후) |
| 패턴 학습 | 자신의 프롬프팅 습관을 인식하고 개선 | 누적 패턴 코칭 |

### 1.3 기능 메카닉 상세

#### 1.3.1 AI 해석 투명화 (Interpretation Transparency)

**목적:** 사용자의 프롬프트를 AI가 어떻게 해석했는지 분해하여, "AI 답변이 왜 이상한지"를 즉시 파악할 수 있게 한다.

**출력 — 3색 해석 리포트:**

| 영역 | 의미 | 예시 |
|------|------|------|
| Referenced (초록) | AI가 프롬프트에서 명확히 식별한 용어·개념 | "React" → JavaScript UI 라이브러리로 해석 |
| Inferred (오렌지) | AI가 직접 받지 않았지만 추론한 배경 맥락 | "웹 애플리케이션" → React 언급에서 추론 (확신: 높음) |
| Missing (빨강) | AI가 모르는 정보, 모호하거나 부재한 것 | "인증 방식" → 이메일/비밀번호? OAuth? SSO? |

**동작 흐름:**

```
사용자 프롬프트 입력 (Claude Desktop/Code에서 자연어로 요청)
    │
    ▼
[MCP Server] analyze_prompt 도구 호출
    │
    ├── local 모드: 규칙 엔진 → 5축 점수 + 누락 요소 + 개선 제안 (무료, 즉시)
    ├── api 모드: Claude API → 3색 리포트 (BYOK, 정밀)
    │
    ▼
[응답] 점수, 누락 요소, 제안, 보강된 프롬프트 반환
```

**분석용 시스템 프롬프트 (api 모드):**

```
사용자의 프롬프트를 분석하여 다음 3가지 카테고리로 분류하라:
1. referenced: 프롬프트에서 명확히 이해한 용어와 개념 (각 항목에 term, explanation)
2. inferred: 명시되지 않았지만 추론한 맥락 (각 항목에 term, explanation, confidence)
3. missing: 부재하거나 모호한 정보 (각 항목에 field, suggestion)
JSON 형식으로 응답하라.
```

**API 응답 스키마:**

```json
{
  "referenced": [{ "term": "...", "explanation": "..." }],
  "inferred": [{ "term": "...", "explanation": "...", "confidence": "high|medium|low" }],
  "missing": [{ "field": "...", "suggestion": "..." }],
  "enhancedPrompt": "..."
}
```

#### 1.3.2 AI 해석 시뮬레이션 (Interpretation Narrative) — v0.4.0

**목적:** 3색 리포트의 분류 목록을 넘어, AI가 1인칭 시점으로 "이 프롬프트를 읽고 어떻게 이해했는지"를 자연어로 서술한다.

**핵심 가치:** 프롬프팅 지식이 없는 일반 사용자도 "아, AI가 여기서 이렇게 추론하는구나"를 즉시 파악. 진입 장벽이 가장 낮고 임팩트가 가장 큰 기능.

**기존 분석과의 차이:**

| 항목 | 기존 (3색 리포트) | 해석 시뮬레이션 |
|------|------------------|---------------|
| 형태 | 분류 목록 | AI 1인칭 서술문 |
| 알려주는 것 | "무엇이 부족한지" | "AI가 어떻게 오해하는지" |
| 필요 지식 | 5축 점수 기준 이해 필요 | 없음 (자연어 읽기만 하면 됨) |

**출력 형식:**

```
[AI의 시점]

나는 이 프롬프트를 받고 당신이 REST API 설계를 원한다고 이해했다.

"시니어 백엔드 개발자"라는 역할 지정이 있어서, 전문적인 수준의 아키텍처 패턴과
실무 용어를 사용하여 답변하겠다.

다만 몇 가지를 직접 추측해야 했다:
- 시스템 규모를 명시하지 않아서 중소규모로 가정했다.
- "좋은 설계"의 기준이 없어서 일반적인 베스트 프랙티스를 나열할 것이다.

출력 형식을 지정하지 않았으므로 자유 서술로 답하겠다.
```

**구현 방식:**

| 모드 | 방식 | 비용 |
|------|------|------|
| api 모드 | Claude API 시스템 프롬프트에 서술형 응답 지시 추가 | API 호출 비용 |
| local 모드 | 규칙 엔진 분석 결과를 템플릿 기반 서술문으로 변환 | 무료 |

**MCP 도구 변경:** `analyze_prompt` 응답에 `narrative` 필드 추가. HistoryEntry에도 `narrative` 저장.

#### 1.3.3 누적 패턴 코칭 (Personal Pattern Coach) — v0.4.0

**목적:** 축적된 히스토리에서 반복 약점 패턴을 추출하고, 개인화된 코칭과 구체적 행동 제안을 제공한다.

**기존 통계와의 차이:**

| 항목 | 기존 (get_stats) | 누적 패턴 코칭 |
|------|-----------------|---------------|
| 데이터 | 평균 점수, 태그 빈도 | 축별 약점 패턴, 도메인별 분석, 성장 추이 |
| 인사이트 | "평균 62점" | "코드 리뷰에서 Format 누락률 78%" |
| 행동 제안 | 없음 | "다음 프롬프트에 '제한사항: ...'을 추가해보세요" |

**출력 — 코칭 리포트:**

```
📊 이번 주 요약
- 분석한 프롬프트: 23개, 평균 점수: 68점 (+6점 ↑)

🔍 반복 패턴
1. 출력 형식 누락 (78%) → 18/23개에서 format 미지정
2. 제약 조건 미명시 (65%)

3. 도메인별 차이
   → 코드 리뷰: Context 82점 (강점), Format 35점 (약점)
   → 데이터 분석: Role 88점 (강점), Example 22점 (약점)

💡 코칭
- "출력 형식: 표 형태로, 열은 [항목, 설명, 우선순위]"를 추가해보세요

📈 성장 추이: W1 52 → W2 58 → W3 62 → W4 68 (+31%)
```

**분석 로직 (local, API 불필요):**

```
get_coaching 호출
    ├── 히스토리 수집 → 축별 통계 → Missing 패턴 빈도
    ├── 도메인별 교차분석 → 성장 추이 계산
    └── 코칭 메시지 생성 (템플릿 기반)
```

**MCP 도구:** `get_coaching` (파라미터: period, projectId). 최소 10개 히스토리 필요.

#### 1.3.4 프롬프트 변이 비교 — Prompt Lab (이후 버전)

같은 의도의 프롬프트를 2~3개 변형하여, "어떤 표현이 AI 해석에 어떤 차이를 만드는지" 비교. 사람은 추상적 조언보다 구체적 비교에서 훨씬 빠르게 학습한다.

#### 1.3.5 응답 품질 감사 — Response Auditor (이후 버전)

AI 답변의 환각, 논리 비약, 출처 불명확 주장을 자동 탐지. 각 마킹 항목에 "이것을 확인하려면..." 후속 질문 제안.

### 1.4 수익 모델

> **현재 (BYOK):** 사용자가 자신의 Claude API 키를 사용하므로 운영 비용 0. 수익화는 이후 버전에서 검토.

| 방식 | 설명 |
|------|------|
| 무료 오픈소스 | BYOK 전용 완전 무료, 커뮤니티 기반 성장 |
| 프리미엄 기능 과금 | Prompt Lab, Response Auditor 등 고급 기능만 유료 |

## 2. 시장 조사

### 2.1 시장 분류

#### 1유형 — 원클릭 프롬프트 개선기 (가장 많음)

| 제품 | 형태 | 핵심 기능 |
|------|------|---------|
| Promptimize | 크롬 확장 | 프롬프트 자동 개선, 변수 템플릿 |
| Prompter | 크롬 확장 | 워크스페이스 컨텍스트 자동 반영 |
| Pretty Prompt | 크롬 확장 + 웹앱 | 원클릭 프롬프트 리파인 |
| Prompt Fixer | 크롬 확장 | 프롬프트 점수 + 개선 |
| PromptPerfect | 웹앱 | 프롬프트 생성 및 최적화 |

공통 한계: 사용자가 "왜" 개선되었는지 학습하지 못함.

#### 2유형 — 분석 + 부족 정보 탐지 (가장 유사)

| 제품 | 형태 | 핵심 기능 |
|------|------|---------|
| FunBlocks AI Prompt Optimizer | 크롬 확장 | 프롬프트 분석, 동적 폼 생성 |
| Prompt Analyzer (GPT) | ChatGPT GPT | LLM 프롬프트 구조/스타일/효과성 피드백 |

FunBlocks 한계: 프롬프트를 "고쳐주는 것"이지 "AI가 어떻게 읽었는지" 보여주는 것은 아님.

#### 3유형 — 점수/효율 측정기

| 제품 | 형태 | 핵심 기능 |
|------|------|---------|
| Promptlee | 크롬 확장 | 효율성 점수(0~100), 토큰 수/비용 추정 |

### 2.2 시장 공백

| 비교 항목 | 기존 툴 | PromptLens |
|---------|---------|-------------|
| 핵심 행위 | 프롬프트를 자동으로 고쳐줌 | AI가 어떻게 읽었는지 보여줌 |
| 부족 정보 | 폼으로 채우게 함 | AI 관점에서 왜 부족한지 설명 |
| 시간 축 | 일회성 | 누적 패턴 코칭 |
| 사용자에게 남는 것 | 더 나은 텍스트 | AI 활용 역량 향상 |

**시장에 없는 기능:** AI 해석 시뮬레이션 (1인칭 서술), 누적 패턴 코칭 (개인화 행동 제안), MCP 네이티브 (대화 중 즉시 사용)

### 2.3 차별화 포지셔닝

```
 자동 개선 ←─────────────────────→ 역량 향상
 │                                           │
 Promptimize      FunBlocks      PromptLens  │
 Pretty Prompt    Prompt         (목표 위치)  │
 Prompt Fixer     Analyzer                    │
 │                                            │
 └──── 확장 도구 ─────────── MCP 도구 ────────┘
```

PromptLens는 **MCP 네이티브 도구**이면서 **AI 활용 역량 향상에 초점**을 맞추는 유일한 포지션.
