## 1. 제품 설계

### 1.1 컨셉

> "AI가 내 말을 어떻게 이해했는지 보여주고, 부족한 맥락을 알아서 채워준다."

현재 상태: ADD 완성 → **AI 확장 툴로 방향 전환**

**제품 방향:** 프롬프트 사고력 훈련 앱에서 **AI 활용 역량을 실질적으로 높여주는 확장 툴**로 피벗. "교육"이 아니라 "즉시 실용" — AI 답변이 왜 이상한지 바로 보여주고, 다음에 뭘 보충하면 되는지 안내하며, 반복되는 맥락은 자동으로 관리해주는 것이 핵심.

**2가지 핵심 축:**

| 축 | 기능 | 사용자 가치 |
|---|------|----------|
| AI 해석 투명화 | 프롬프트를 넣으면 AI가 뭘 이해했고 뭘 모르는지 시각화 | "AI 답변이 왜 이상한지" 즉시 파악 |
| 맥락 빌더 | 프로젝트·역할·선호 스타일을 프로필로 저장, 대화에 필요한 맥락만 지능적으로 선별·삽입 | 매번 배경 설명 반복 불필요 |

**보조 기능 (확장 로드맵):**

| 기능 | 설명 |
|------|------|
| 결과 비교 (Prompt Lab) | 같은 의도의 프롬프트 2~3개를 동시 실행, 결과 나란히 비교 |
| 응답 품질 감사 (Response Auditor) | AI 답변의 환각, 논리 비약, 출처 불명확 주장 자동 탐지 |
| 워크플로우 패턴 | 작업별 최적 AI 활용 절차 (기획서, 코드 리뷰, 데이터 분석 등) |

**기술 스택:**

| 계층 | 기술 | 상태 |
|------|------|------|
| 코어 | TypeScript 순수 패키지 (@promptcraft/core) | 확정 |
| 클라이언트 1 | 크롬 확장 (React + Vite/CRXJS) | 확정 (Phase 1) |
| 클라이언트 2 | 데스크톱 앱 (Tauri 2.0 + React) | 확정 (Phase 2) |
| 백엔드 | AWS Lambda + API Gateway (서버리스) | 확정 |
| AI 연동 | Anthropic Claude API | 확정 |

### 1.2 핵심 철학 — AI 활용 역량 5가지

사고력 훈련만으로는 부족하다. 사용자가 AI를 더 잘 쓰려면 **5가지 역량**이 필요하고, PromptCraft는 이를 도구 사용 과정에서 자연스럽게 키워준다.

| 역량 | 정의 | PromptCraft가 돕는 방식 |
|------|------|---------------------|
| AI 해석 이해 | AI가 내 말을 어떻게 읽는지 아는 것 | 해석 투명화 — 참고 용어, 추론 맥락, 부재 정보 시각화 |
| 맥락 관리 | AI에게 필요한 배경 정보를 효율적으로 전달 | 맥락 빌더 — 프로필 저장 + 지능형 맥락 선별·삽입 |
| 결과 검증 | AI 답변의 품질과 신뢰도를 판단 | 응답 감사 — 환각, 논리 비약, 근거 부족 탐지 |
| 실험적 접근 | 다양한 표현으로 시도하고 비교하는 습관 | Prompt Lab — 동시 실행 + 결과 비교 |
| 워크플로우 설계 | 복잡한 작업을 AI와 협업하는 절차를 만드는 능력 | 워크플로우 패턴 라이브러리 |

**기존 3대 사고 능력과의 관계:**

```
기존 (교육 중심)              →  새 방향 (실용 중심)
─────────────────────────────────────────────────
자기 명확화                   →  AI 해석 이해 + 맥락 관리
인지적 탈중심화                →  AI 해석 이해 (AI 관점 체험)
역추적 사고                   →  결과 검증 + 실험적 접근
(없음)                       →  워크플로우 설계 (신규)
```

### 1.3 기능 메카닉 상세

#### 1.3.1 AI 해석 투명화 (Interpretation Transparency)

**목적:** 사용자의 프롬프트를 AI가 어떻게 해석했는지 분해·시각화하여, "AI 답변이 왜 이상한지"를 즉시 파악할 수 있게 한다.

**입력:** 사용자 프롬프트 텍스트 (직접 입력 또는 AI 채팅창에서 캡처)

**출력 — 3색 해석 리포트:**

| 영역 | 의미 | 표시 | 예시 |
|------|------|------|------|
| Referenced (초록) | AI가 프롬프트에서 명확히 식별한 용어·개념 | 초록 좌측 바 + 항목 리스트 | "React" → JavaScript UI 라이브러리로 해석 |
| Inferred (오렌지) | AI가 직접 받지 않았지만 스스로 추론한 배경 맥락 | 오렌지 좌측 바 + 확신도(높음/보통/낮음) | "웹 애플리케이션" → React 언급에서 추론 (확신: 높음) |
| Missing (빨강) | AI가 모르는 정보, 모호하거나 부재한 것 | 빨강 좌측 바 + 보충 제안 | "인증 방식" → 이메일/비밀번호? OAuth? SSO? |

**동작 흐름:**

```
사용자 프롬프트 입력
    │
    ▼
[클라이언트] POST /api/analyze { prompt, context? }
    │
    ▼
[백엔드 Lambda] Claude API 호출 (해석 분석 전용 시스템 프롬프트)
    │
    ▼
[백엔드] JSON 응답 파싱 → referenced / inferred / missing 분류
    │
    ▼
[클라이언트] 3색 리포트 렌더링
    │
    ├─ Missing 항목 중 프로필 매칭 → autoFillable: true 표시
    │
    ▼
[클라이언트] 보강된 프롬프트(enhancedPrompt) 생성 및 미리보기
```

**분석용 시스템 프롬프트:**

```
사용자의 프롬프트를 분석하여 다음 3가지 카테고리로 분류하라:
1. referenced: 프롬프트에서 명확히 이해한 용어와 개념 (각 항목에 term, explanation)
2. inferred: 명시되지 않았지만 추론한 맥락 (각 항목에 term, explanation, confidence)
3. missing: 부재하거나 모호한 정보 (각 항목에 field, suggestion)
JSON 형식으로 응답하라.
```

**API 스키마:**

```
Request:  { prompt: string, context?: ContextProfile, platform?: string }
Response: {
  referenced: [{ term, explanation }],
  inferred:   [{ term, explanation, confidence: "high"|"medium"|"low" }],
  missing:    [{ field, suggestion, autoFillable, profileValue? }],
  enhancedPrompt: string
}
```

#### 1.3.2 맥락 빌더 (Context Builder)

**목적:** 사용자의 프로젝트·역할·기술 스택·선호 스타일을 프로필로 저장하고, AI 대화 시 필요한 맥락을 자동으로 선별·삽입하여 매번 배경 설명을 반복하지 않게 한다.

**프로필 데이터 모델:**

```
ContextProfile {
  id: string
  name: string              // "웹 개발 프로젝트", "마케팅 업무"
  role: string              // "시니어 백엔드 개발자"
  domain: string            // "핀테크", "에듀테크"
  techStack: string[]       // ["React", "TypeScript", "Tailwind"]
  preferences: string       // "코드는 TypeScript, 설명은 한국어"
  customContext: string     // 자유 입력 맥락
  isActive: boolean
  createdAt: timestamp
  updatedAt: timestamp
}
```

**맥락 삽입 형식 (AI에 전달되는 포맷):**

```
[PromptCraft Context]
역할: {role}
도메인: {domain}
기술 스택: {techStack.join(', ')}
선호: {preferences}
추가 맥락: {customContext}
[/PromptCraft Context]

{사용자 원본 프롬프트}
```

**동작 흐름:**

```
[설정] 사용자가 프로필 생성 (이름, 역할, 도메인, 기술 스택, 선호, 추가 맥락)
    │
    ▼
[설정] 활성 프로필 1개 선택 (토글)
    │
    ▼
[사용 시] 프롬프트 입력 → 분석 요청 시 활성 프로필 맥락이 자동 첨부
    │
    ▼
[분석 결과] Missing 항목 중 프로필로 채울 수 있는 것 자동 매칭
    │
    ├─ 매칭 성공 → "프로필에서 자동 보충: Tailwind" (초록 뱃지)
    ├─ 매칭 실패 → 사용자에게 보충 가이드 제공
    │
    ▼
[출력] 보강된 프롬프트 = 맥락 블록 + 원본 프롬프트 + 자동 보충 조건
```

**자동 보충 매칭 로직:**

| Missing 필드 | 프로필에서 탐색하는 곳 | 예시 |
|-------------|-------------------|------|
| 스타일링 방식 | techStack 중 CSS 프레임워크 | "Tailwind" |
| 상태 관리 | techStack 중 상태 관리 라이브러리 | "Zustand" |
| 데이터베이스 | techStack 중 DB | "PostgreSQL" |
| 대상 사용자 | domain 필드 | "에듀테크 분야 사용자" |
| 배포 환경 | techStack 중 클라우드 | "AWS" |

#### 1.3.3 해석 + 맥락 통합 루프

두 핵심 기능이 하나의 루프로 연결되어 PromptCraft의 핵심 사용 경험을 형성한다.

```
┌─────────────────────────────────────────────────────────┐
│  ① 프롬프트 입력                                          │
│     사용자가 AI 채팅창에서 프롬프트 작성                      │
│                                                          │
│  ② 해석 투명화 실행                                       │
│     → 3색 리포트: Referenced 4건 / Inferred 3건 / Missing 5건 │
│                                                          │
│  ③ 맥락 빌더 자동 연동                                    │
│     → Missing 5건 중 2건은 프로필에서 자동 보충 가능           │
│     → 나머지 3건은 사용자에게 보충 가이드 제공                 │
│                                                          │
│  ④ 보강된 프롬프트 생성                                    │
│     → [맥락 블록] + [원본 프롬프트] + [자동 보충 조건]         │
│     → 사용자가 미리보기 확인                                 │
│                                                          │
│  ⑤ AI에 전송                                             │
│     → "프롬프트 복사" 또는 "AI에 전송" 버튼                  │
│     → 사용자가 AI 채팅창에 붙여넣기 또는 자동 삽입             │
└─────────────────────────────────────────────────────────┘
```

**핵심 지표:** 이 루프를 완주하는 비율(분석→보충→전송)이 60% 이상이어야 MVP 성공으로 판단한다.

#### 1.3.4 Prompt Lab (확장 — Phase 2+)

**목적:** 같은 의도의 프롬프트를 2~3개 변형하여 동시 실행하고, 결과를 나란히 비교하여 "어떤 표현이 더 나은 결과를 만드는지" 실험적으로 학습한다.

**메카닉:**

```
[입력] 원본 프롬프트 A
    │
    ├── 사용자가 직접 변형 B 작성
    ├── AI가 자동 변형 C 제안 (표현 변경, 제약 추가 등)
    │
    ▼
[실행] A, B, C를 동일 AI 모델에 동시 전송
    │
    ▼
[비교] 결과 3개를 나란히 표시
    │
    ├── 차이점 하이라이트 (어떤 부분이 다르게 나왔는지)
    ├── 각 결과에 대한 품질 점수 (선택)
    │
    ▼
[학습] "왜 이 변형이 더 나은 결과를 만들었는지" 설명
```

#### 1.3.5 Response Auditor (확장 — Phase 2+)

**목적:** AI 답변의 환각(hallucination), 논리 비약, 출처 불명확 주장을 자동 탐지하여, 사용자가 AI 답변을 무비판적으로 수용하지 않도록 돕는다.

**메카닉:**

```
[입력] AI의 응답 텍스트
    │
    ▼
[분석] Claude API로 검증 분석 요청 (별도 시스템 프롬프트)
    │
    ▼
[출력] 3가지 마킹
    │
    ├── 환각 의심 (빨강) — 사실과 다를 가능성이 높은 주장
    │   예: "React 19에서 Server Components가 기본값이 됨" → 확인 필요
    │
    ├── 논리 비약 (오렌지) — 전제에서 결론까지 빠진 단계가 있는 부분
    │   예: "따라서 마이크로서비스가 최선입니다" → 단일 서비스와 비교 없음
    │
    ├── 근거 부족 (노랑) — 출처나 구체적 데이터 없이 단정하는 부분
    │   예: "대부분의 개발자가 선호합니다" → 어떤 조사?
    │
    ▼
[가이드] 각 마킹 항목에 대해 "이것을 확인하려면..." 후속 질문 제안
```

#### 1.3.6 워크플로우 패턴 (확장 — Phase 3+)

**목적:** 반복되는 작업(기획서 작성, 코드 리뷰, 데이터 분석 등)에 대해 AI와 협업하는 최적 절차를 템플릿화하여, 사용자가 처음부터 설계하지 않아도 되게 한다.

**메카닉:**

```
[패턴 라이브러리]
    │
    ├── 기획서 작성 패턴
    │   Step 1: 목적·대상 정의 (맥락 빌더 프로필 활용)
    │   Step 2: 구조 초안 생성 (AI)
    │   Step 3: 각 섹션 상세화 (해석 투명화로 검증)
    │   Step 4: 최종 검토 (Response Auditor로 감사)
    │
    ├── 코드 리뷰 패턴
    │   Step 1: 코드 + 요구사항 맥락 전달
    │   Step 2: AI 리뷰 요청 (버그, 성능, 가독성)
    │   Step 3: 리뷰 결과 감사 (환각 탐지)
    │   Step 4: 수정사항 적용 + 재검증
    │
    ├── 사용자 커스텀 패턴
    │   사용자가 직접 Step 정의 → 저장 → 재사용
    │
    ▼
[실행] 패턴 선택 → Step별 가이드에 따라 진행 → 완료 후 결과 요약
```

### 1.4 핵심 인터랙션 루프 (5단계)

```
1. 자기 명확화  →  "나는 지금 정확히 뭘 원하지?"
2. 간극 인식    →  "AI가 이걸 읽으면 뭘 모를까?"
3. 작성        →  실제 프롬프트 쓰기
4. 결과 관찰   →  AI 응답 확인
5. 역추적      →  "왜 이렇게 나왔지? 뭐가 부족했지?"
   → 개선 → 1단계로 복귀
```

### 1.5 주요 모드

**씽킹 빌더 (Thinking Builder) — 메인**

- 1단계: 유도 질문으로 의도 정리
- 2단계: AI가 모를 정보 목록 제시 → 사용자가 보충 여부 판단
- 3단계: 자유 작성 + 선택적 블록 가이드 (목표/맥락/역할/제약/출력/예시)
- 4단계: AI 응답을 초록/빨강으로 주석 평가
- 5단계: 빨강 영역의 원인을 프롬프트에서 역추적

**챌린지 모드 — 서브**

- 유형 A: 결과 매칭 (목표 결과를 얻는 프롬프트 작성)
- 유형 B: 의도 추론 (결과만 보고 원래 프롬프트 추측)
- 유형 C: 간극 탐지 (프롬프트에서 누락 정보 찾기)
- 유형 D: 역추적 퀴즈 (결과와 프롬프트 보고 원인 매칭)

### 1.6 평가 & 성장

**평가 비중:**

| 항목 | 비중 |
|------|------|
| 사고 과정 점수 (1~2단계 + 5단계) | 55% |
| 프롬프트 품질 (3단계) | 25% |
| 결과 유사도 (4단계) | 20% |

**성장 시스템 (ADD 용어):** 숙련도/숙련 단계, 마일스톤, 커뮤니티 순위, 입문~전문가 난이도, 가이드, 명칭

## 2. 시장 조사 (2026-03-20)

### 2.1 시장 분류

#### 1유형 — 원클릭 프롬프트 개선기 (가장 많음)

사용자가 프롬프트를 쓰면 자동으로 개선된 버전을 생성해주는 도구.

| 제품 | 형태 | 핵심 기능 | 특징 |
|------|------|---------|------|
| Promptimize | 크롬 확장 | 프롬프트 자동 개선, 변수 템플릿 | "Grammarly for prompting" 표방, 모든 주요 AI 플랫폼 지원 |
| Prompter | 크롬 확장 | 워크스페이스 컨텍스트 자동 반영 | 프로젝트 정보를 저장하면 매번 프롬프트에 자동 삽입 |
| Pretty Prompt | 크롬 확장 + 웹앱 | 원클릭 프롬프트 리파인 | 역할 할당, 출력 포맷, 제약조건을 자동으로 추가 |
| Prompt Fixer | 크롬 확장 | 프롬프트 점수 + 개선 | ChatGPT, Claude, Gemini, Perplexity 지원 |
| PromptPerfect | 웹앱 | 프롬프트 생성 및 최적화 | GPT-4, ChatGPT, Midjourney 등 다양한 모델 지원 |

공통 한계: 사용자가 "왜" 개선되었는지 학습하지 못함. 도구 의존도가 높아짐.

#### 2유형 — 분석 + 부족 정보 탐지 (가장 유사)

| 제품 | 형태 | 핵심 기능 | 특징 |
|------|------|---------|------|
| FunBlocks AI Prompt Optimizer | 크롬 확장 | 프롬프트 분석, 동적 폼 생성, 비판적 사고 도구 | 누락 정보 탐지 시 맞춤 폼 자동 생성 |
| FunBlocks Critical Response Analyzer | 크롬 확장 (위와 통합) | AI 응답의 논리적 일관성, 편향, 대안적 관점 평가 | 관련 질문과 연결 주제 탐색 |
| Prompt Analyzer (GPT) | ChatGPT GPT | LLM 프롬프트 구조/스타일/효과성 피드백 | ChatGPT Plus 필요 |

**FunBlocks 상세 (가장 가까운 경쟁자):** 프롬프트 최적화, 동적 폼(누락 정보 → 필드 생성 → 사용자 채움 → 통합), 질문 최적화(1→5 변환), 비판적 분석, 심층 탐색. 지원: ChatGPT, Claude, Gemini, Perplexity, DeepSeek. 가격: 신규 30회 + 일 10회 무료.

FunBlocks 한계: 프롬프트를 "고쳐주는 것"이지 "AI가 뭘 이해하고 뭘 못 이해했는지" 보여주는 것은 아님.

#### 3유형 — 점수/효율 측정기

| 제품 | 형태 | 핵심 기능 | 특징 |
|------|------|---------|------|
| Promptlee | 크롬 확장 | 효율성 점수(0~100), 토큰 수/비용/에너지 추정 | 규칙 기반 분석 (AI 토큰 미소비), CSV 내보내기 |
| MaxAI Prompt Enhancer | 웹앱 | 단계별 프롬프트 분석 및 피드백 | 반복적 개선 가이드 제공 |

### 2.2 시장 공백

**기존 툴 vs PromptCraft:**

| 비교 항목 | 기존 툴 (1~3유형) | PromptCraft |
|---------|--------------|-------------|
| 핵심 행위 | 프롬프트를 자동으로 고쳐줌 | AI가 어떻게 읽었는지 보여줌 + 맥락 자동 관리 |
| 부족 정보 처리 | 폼으로 채우게 함 | 왜 부족한지 AI 관점에서 설명 + 프로필에서 자동 보충 |
| 결과 검증 | 없음 | 환각·논리 비약·근거 부족 자동 탐지 |
| 사용자에게 남는 것 | 더 나은 프롬프트 텍스트 | AI 활용 역량 전반의 향상 |
| 학습 방식 | 없음 (도구 의존) | 도구 사용 과정에서 자연스럽게 체득 |

**시장에 없는 기능:**

1. **AI 해석 시각화** — AI가 어떤 용어를 참고하고, 어떤 맥락을 추론하고, 어떤 정보가 부재한지 분해하여 시각화
2. **지능형 맥락 관리** — 단순 삽입이 아니라 "이 대화에서 필요한 맥락만 선별"하는 프로필 기반 맥락 빌더
3. **결과 비교 실험** — 일반 사용자가 프롬프트 변형을 동시 실행하고 나란히 비교하는 도구
4. **응답 품질 감사** — AI 답변의 환각, 논리 비약, 출처 불명확을 구체적으로 마킹

### 2.3 차별화 & 포지셔닝

| 차별점 | 설명 |
|--------|------|
| AI 해석 투명화 | "AI는 이 프롬프트를 이렇게 읽었습니다" — 참고 용어, 추론 맥락, 부재 정보 시각화 |
| 지능형 맥락 관리 | 프로필 기반으로 대화에 필요한 맥락만 선별·삽입, 매번 배경 설명 반복 불필요 |
| 결과 검증 & 실험 | AI 답변 품질 감사 + 프롬프트 변형 동시 비교 |
| 자연스러운 역량 성장 | 도구를 쓰는 과정에서 AI 활용 역량 5가지가 자연스럽게 체득됨 |

**포지셔닝 맵:**

```
 자동 개선 ←──────────────────→ 역량 향상
 │                                            │
 ┌────────────────┤            ├────────────────┐
 │                │            │                │
 Promptimize      FunBlocks    PromptCraft      │
 Pretty Prompt    Prompt       (목표 위치)       │
 Prompt Fixer     Analyzer                      │
 │                                              │
 └──── 확장 도구 ──────────────── 독립 앱 ───────┘
```

PromptCraft는 **확장 도구 형태**이면서 **AI 활용 역량 향상에 초점**을 맞추는 유일한 포지션. "사고력 훈련"이 아니라 "실용적 도구를 쓰면서 자연스럽게 역량이 오르는" 경험을 제공한다.
