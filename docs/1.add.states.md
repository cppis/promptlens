# PromptLens MCP — 명령어 및 상태 변환 정리

## 1. 명령어 목록 (18개)

### 프로젝트 관리 (4개)

| 명령어 | 파라미터 | 설명 |
|--------|----------|------|
| `list_projects` | — | 전체 프로젝트 목록 + 통계 (prompt 수, 평균 점수, 트렌드) |
| `create_project` | `name` | 새 프로젝트 생성 |
| `set_active_project` | `action` (set/clear/get), `projectName` | 활성 프로젝트 설정/해제/조회 |
| `snapshot_project` | `projectId`, `label?`, `compareWith?` | 프로젝트 스냅샷 생성 및 비교 |

### 분석 및 평가 (3개)

| 명령어 | 파라미터 | 설명 |
|--------|----------|------|
| `analyze_prompt` | `prompt`, `mode?`, `autoRun?`, `parentId?`, `projectId?`, `tags?` | 프롬프트 5축 분석 (clarity, completeness, specificity, context, output_format) |
| `compare_prompts` | `entryIdA`, `entryIdB`, `projectId?` | 두 버전의 텍스트/점수/태그 변화 비교 |
| `get_versions` | `entryId`, `projectId?` | 프롬프트의 전체 버전 체인 조회 |

### 히스토리 관리 (3개)

| 명령어 | 파라미터 | 설명 |
|--------|----------|------|
| `add_history_entry` | `projectId`, `prompt`, `score?`, `tags?`, `note?` | 수동으로 프롬프트 기록 추가 |
| `get_history` | `projectId?`, `limit?`, `pageToken?`, `search?` | 페이지네이션 기반 히스토리 조회 |
| `query_history` | `projectId?`, `limit?`, `scoreMin/Max?`, `grade?`, `tags?`, `dateFrom/To?`, `search?`, `sortBy?`, `missing?` | 고급 필터링 히스토리 검색 |

### 시각화 및 내보내기 (3개)

| 명령어 | 파라미터 | 설명 |
|--------|----------|------|
| `visualize_project` | `projectId?`, `outputPath?` | HTML 대시보드 생성 (차트 포함) |
| `export_project` | `projectId?`, `format?`, `dateFrom/To?`, `scoreMin/Max?`, `grade?`, `tags?`, `outputPath?` | JSON/Markdown/CSV 내보내기 |
| `import_project` | `filePath`, `mode?`, `projectName?` | `.promptlens.json` 파일 가져오기 |

### 설정 및 통계 (3개)

| 명령어 | 파라미터 | 설명 |
|--------|----------|------|
| `get_settings` | — | API 키 상태, 모델, 활성 프로젝트, 저장 경로 조회 |
| `set_api_key` | `apiKey`, `model?` | Anthropic API 키 등록 (api 모드 활성화) |
| `get_stats` | — | 전체 통계: 프로젝트 수, 총 프롬프트 수, 평균 점수, 트렌드 |

### 통합 (2개)

| 명령어 | 파라미터 | 설명 |
|--------|----------|------|
| `import_claude_conversations` | `filePath`, `mode?`, `projectName?` | Claude Desktop/Code 대화 가져오기 |
| `get_stats` | — | 전체 통계 조회 |

## 2. 컨텍스트 상태 변환 다이어그램

### 2.1 프로젝트 생명주기

```
┌─────────────┐   create_project   ┌──────────────┐  set_active_project  ┌─────────────────┐
│  (없음)      │ ────────────────→  │  프로젝트     │ ──────────────────→  │  활성 프로젝트    │
│             │                    │  (비활성)     │                      │  (auto-save ON)  │
└─────────────┘                    └──────────────┘  ←──────────────────  └─────────────────┘
                                         ↑            set_active_project         │
                                         │            (action: "clear")          │
                                         │                                      │
                                         │         snapshot_project              │
                                         │  ←───────────────────────────────────┘
                                         │         export_project
                                         │         visualize_project
                                         │         import_project
                                         └──────── (프로젝트에 데이터 축적)
```

## 3. 전제 조건 요약

| 명령어 | 전제 조건 |
|--------|----------|
| `analyze_prompt` | 프롬프트 텍스트 필수. `api`/`deep` 모드는 API 키 필요. `parentId` 지정 시 해당 엔트리 존재 필요 |
| `compare_prompts` | 두 entryId 모두 존재해야 함 |
| `get_versions` | 해당 entryId 존재 필요 |
| `set_active_project` (set) | 프로젝트가 이미 생성되어 있어야 함 |
| `export_project` | 프로젝트에 1개 이상의 엔트리 존재 필요 |
| `import_project` | 유효한 `.promptlens.json` 파일 필요 |
| `set_api_key` | 유효한 Anthropic API 키 필요 |
| `snapshot_project` | `compareWith` 사용 시 이전 스냅샷 존재 필요 |

## 4. 트리거 단축 명령 (>> 접미사)

| 트리거 | 동작 |
|--------|------|
| `>> anz` | `analyze_prompt` local 모드 실행 |
| `>> anz+run` | 분석 + 개선된 프롬프트 즉시 실행 |
| `>> deep` | deep 모드 전체 분석 파이프라인 |
| `>> diff` | 최근 두 엔트리 `compare_prompts` |
| `>> viz` | 활성 프로젝트 `visualize_project` |
