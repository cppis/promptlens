## 1. 프로토타입 목표

### 1.1 목적

핵심 가설 2가지를 최소 비용으로 검증한다.

| 가설 | 검증 질문 |
|------|---------|
| AI 해석 투명화 | 사용자가 "AI가 내 프롬프트를 어떻게 읽었는지" 시각화를 보고 프롬프트를 개선할 수 있는가? |
| 맥락 빌더 | 저장된 프로필 맥락이 자동 삽입되면 AI 답변 품질이 체감할 수 있을 만큼 올라가는가? |

### 1.2 프로토타입 범위

**포함 (MVP):**

- AI 해석 투명화 (핵심 축 1)
- 맥락 빌더 기본 (핵심 축 2)
- 크롬 확장 팝업 UI

**제외 (이후 버전):**

- Prompt Lab (결과 비교)
- Response Auditor (응답 품질 감사)
- 워크플로우 패턴 라이브러리
- 사용자 계정/인증
- 수익화 (프리미엄 기능)

## 2. MVP 기능 범위

### 2.1 기능 A — AI 해석 투명화

사용자가 프롬프트를 입력하면, AI가 해당 프롬프트를 어떻게 해석했는지 분해하여 보여준다.

**입력:** 사용자 프롬프트 텍스트 (직접 입력 또는 AI 채팅창에서 캡처)

**출력 — 해석 리포트 3영역:**

| 영역 | 설명 | 표시 방식 |
|------|------|---------|
| 참고 용어 (Referenced) | AI가 이해한 핵심 키워드·개념 목록 | 초록색 하이라이트 |
| 추론 맥락 (Inferred) | AI가 명시적으로 받지 않았지만 스스로 추론한 배경 | 노란색 하이라이트 + 추론 근거 표시 |
| 부재 정보 (Missing) | AI가 모르는 정보, 모호한 부분 | 빨간색 하이라이트 + 보충 제안 |

**동작 흐름:**

```
1. 사용자가 프롬프트 입력 (또는 채팅창에서 캡처)
2. Claude API로 해석 분석 요청
3. 응답을 3영역으로 파싱
4. 시각화된 해석 리포트 표시
5. 부재 정보 항목 클릭 → 보충 가이드 제공
```

**Claude API 프롬프트 설계 (해석 분석용):**

```
시스템 프롬프트:
"사용자의 프롬프트를 분석하여 다음 3가지 카테고리로 분류하라:
 1. referenced: 프롬프트에서 명확히 이해한 용어와 개념
 2. inferred: 명시되지 않았지만 추론한 맥락 (추론 근거 포함)
 3. missing: 부재하거나 모호한 정보 (보충 제안 포함)
JSON 형식으로 응답하라."
```

### 2.2 기능 B — 맥락 빌더

사용자의 프로젝트, 역할, 선호 스타일 등을 프로필로 저장하고, AI 대화 시 필요한 맥락을 자동으로 선별·삽입한다.

**프로필 데이터 모델:**

```
ContextProfile {
  id: string
  name: string              // "웹 개발 프로젝트", "마케팅 업무" 등
  role: string              // "시니어 백엔드 개발자", "콘텐츠 마케터" 등
  domain: string            // "핀테크", "에듀테크", "이커머스" 등
  techStack: string[]       // ["React", "Node.js", "PostgreSQL"]
  preferences: string       // "코드는 TypeScript, 설명은 한국어, 간결하게"
  customContext: string     // 자유 입력 맥락
  isActive: boolean
  createdAt: timestamp
  updatedAt: timestamp
}
```

**동작 흐름:**

```
1. 사용자가 프로필 1개 이상 생성 (설정 화면)
2. 활성 프로필 선택
3. AI 채팅창에서 프롬프트 작성 시:
   a. 확장 아이콘 클릭 또는 단축키
   b. 활성 프로필의 맥락을 프롬프트 앞에 자동 삽입
   c. (선택) 해석 투명화와 연동 — 부재 정보 중 프로필로 채울 수 있는 항목 자동 보충
```

**맥락 삽입 형식:**

```
[PromptCraft Context]
역할: {role}
도메인: {domain}
기술 스택: {techStack}
선호: {preferences}
추가 맥락: {customContext}
[/PromptCraft Context]

{사용자 원본 프롬프트}
```

### 2.3 기능 연동 — 해석 + 맥락 루프

두 핵심 기능이 하나의 루프로 연결된다.

```
┌─ 프롬프트 입력 ─────────────────────────────────┐
│                                                 │
│  ① 해석 투명화 실행                               │
│     → 부재 정보 3건 발견                          │
│                                                 │
│  ② 맥락 빌더 연동                                │
│     → 부재 정보 중 2건은 프로필에서 자동 보충 가능    │
│     → 1건은 사용자에게 보충 가이드 제공              │
│                                                 │
│  ③ 보강된 프롬프트 생성                            │
│     → 사용자 확인 후 AI에 전송                     │
└─────────────────────────────────────────────────┘
```

## 3. 기술 아키텍처

### 3.1 설계 원칙 — 멀티 플랫폼

코어 로직을 플랫폼에 독립적인 패키지로 분리하여, **크롬 확장**과 **데스크톱 앱** 모두에서 동일한 기능을 제공한다.

```
┌─────────────────────────────────────────────────────────┐
│                      클라이언트 (Shell)                    │
│                                                          │
│   ┌──── 크롬 확장 ────┐    ┌──── 데스크톱 앱 ────┐        │
│   │  Popup UI (React) │    │  Window UI (React)  │        │
│   │  Content Script   │    │  Electron / Tauri   │        │
│   │  Service Worker   │    │  System Tray        │        │
│   └────────┬──────────┘    └────────┬────────────┘        │
│            │                        │                     │
│   ┌────────┴────────────────────────┴──────────┐         │
│   │              @promptcraft/core              │         │
│   │  AnalyzerService · ProfileManager           │         │
│   │  ContextBuilder · StorageAdapter(interface)  │         │
│   └──────────────────┬──────────────────────────┘         │
└──────────────────────┼────────────────────────────────────┘
                       │ HTTPS
              ┌────────┴────────┐
              │   백엔드 API     │
              │  (서버리스)       │
              └────────┬────────┘
                       │
              ┌────────┴────────┐
              │  Claude API      │
              │  (Anthropic)     │
              └─────────────────┘
```

**핵심:** `@promptcraft/core`는 순수 TypeScript 패키지로 UI·저장소·플랫폼 API에 의존하지 않는다. 각 플랫폼 Shell이 StorageAdapter, ClipboardAdapter 등을 주입(DI)한다.

### 3.2 멀티 플랫폼 비교

| 항목 | 크롬 확장 | 데스크톱 앱 |
|------|---------|----------|
| 배포 | Chrome Web Store | 직접 배포 (GitHub Releases / 자체 업데이터) |
| UI 프레임워크 | React (팝업 + 사이드패널) | React (Electron / Tauri 윈도우) |
| 저장소 | Chrome Storage API | SQLite (로컬) + 파일 시스템 |
| AI 채팅 연동 | Content Script (DOM 조작) | 클립보드 감시 + 글로벌 단축키 |
| 시스템 연동 | 제한적 (브라우저 탭 내) | 트레이 아이콘, 글로벌 단축키, 알림 |
| 오프라인 | 불가 | 로컬 분석 캐시 가능 |
| 장점 | 설치 간편, AI 채팅과 직접 연동 | 플랫폼 제약 없음, 더 넓은 UI, 시스템 레벨 통합 |
| 단점 | 크롬 전용, 팝업 크기 제한 | 별도 설치 필요, 채팅 연동이 간접적 |

### 3.3 기술 스택

**공통 (Core):**

| 계층 | 기술 | 선택 이유 |
|------|------|---------|
| 코어 로직 | TypeScript (순수 패키지) | 플랫폼 독립, 타입 안전 |
| UI 프레임워크 | React + TypeScript | 크롬 확장 & 데스크톱 모두 사용 |
| 스타일링 | Tailwind CSS | 유틸리티 기반, 양쪽 플랫폼 동일 스타일 |
| 백엔드 API | AWS Lambda + API Gateway | 서버리스, 초기 비용 최소화 |
| AI 연동 | Claude API (claude-sonnet-4-6) | 해석 분석에 적합한 성능/비용 밸런스 |

**크롬 확장 Shell:**

| 계층 | 기술 | 선택 이유 |
|------|------|---------|
| 빌드 | Vite + CRXJS | HMR 지원, Manifest V3 빌드 최적화 |
| 저장소 | Chrome Storage API | 프로필 데이터 저장, 디바이스 간 동기화 |
| AI 채팅 연동 | Content Script | claude.ai, chatgpt.com DOM 직접 조작 |

**데스크톱 Shell:**

| 계층 | 기술 | 선택 이유 |
|------|------|---------|
| 앱 프레임워크 | Tauri 2.0 (Rust + WebView) | 경량 (Electron 대비 ~10x 작은 바이너리), 네이티브 성능 |
| 빌드 | Vite | 크롬 확장과 동일 빌드 도구 |
| 저장소 | SQLite (tauri-plugin-sql) | 구조화된 로컬 저장, 오프라인 지원 |
| AI 채팅 연동 | 클립보드 감시 + 글로벌 단축키 | 플랫폼 무관하게 동작 |
| 시스템 통합 | System Tray + 글로벌 단축키 | 항상 대기, 빠른 접근 |

### 3.5 어댑터 인터페이스

```typescript
// IStorageAdapter.ts
interface IStorageAdapter {
  get<T>(key: string): Promise<T | null>;

  getAll(): Promise<Record<string, unknown>>;

  read(): Promise<string>;

}
```

### 3.6 데스크톱 앱 — AI 채팅 연동 방식

크롬 확장은 Content Script로 AI 채팅창에 직접 접근하지만, 데스크톱 앱은 간접 방식을 사용한다.

**방식 A — 클립보드 기반 (기본):**

```
1. 사용자가 AI 채팅창에서 프롬프트를 작성
2. 글로벌 단축키 (Ctrl+Shift+A) 입력
3. 데스크톱 앱이 클립보드에서 선택된 텍스트 읽기
4. 해석 분석 실행 → 결과를 플로팅 윈도우로 표시
5. "보강 프롬프트 복사" → 클립보드에 복사 → 사용자가 붙여넣기
```

**방식 B — 동반 크롬 확장 (선택):**

```
1. 경량 크롬 확장이 WebSocket으로 데스크톱 앱과 통신
2. Content Script가 프롬프트 캡처 → 데스크톱 앱으로 전송
3. 데스크톱 앱에서 분석 → 결과를 크롬 확장으로 반환
4. 크롬 확장이 채팅창 옆에 인라인 결과 표시
```

### 3.8 API 설계

**POST /api/analyze — 해석 분석**

```json
// Request
{
  "prompt": "React로 로그인 폼 만들어줘",
  "context": { ... },  // 선택: 활성 프로필 맥락
  "platform": "claude",
  "clientType": "chrome-extension" | "desktop"
}

// Response
{
  "referenced": [
    { "term": "React", "explanation": "JavaScript UI 라이브러리로 해석" },
    { "term": "로그인 폼", "explanation": "사용자 인증 입력 UI로 해석" }
  ],
  "inferred": [
    { "term": "웹 애플리케이션", "explanation": "React 언급으로 웹 환경 추론", "confidence": "high" },
    { "term": "함수형 컴포넌트", "explanation": "최신 React 관행 추론", "confidence": "medium" }
  ],
```

## 5. 개발 일정

### 5.1 전체 일정 (8주, 2단계)

**Phase 1 — 크롬 확장 (1~6주):**

| 주차 | 단계 | 산출물 |
|------|------|--------|
| 1주 | 모노레포 셋업 + 코어 패키지 | pnpm workspace, @promptcraft/core, 어댑터 인터페이스, shared-ui 스캐폴딩 |
| 2주 | 크롬 확장 Shell + 기본 UI | Manifest V3, 팝업 라우팅, ChromeStorageAdapter, 프로필 CRUD UI |
| 3주 | 해석 투명화 코어 | Claude API 연동, AnalyzerService, 해석 프롬프트 튜닝, 3색 리포트 UI |
| 4주 | 맥락 빌더 코어 | ProfileManager, ContextBuilder, Content Script (claude.ai), 맥락 삽입 |
| 5주 | 기능 연동 + ChatGPT 지원 | 해석↔맥락 루프, 보강 프롬프트, Content Script (chatgpt.com), 테스트 |
| 6주 | 크롬 확장 베타 배포 | Chrome Web Store 등록, 피드백 수집 |

**Phase 2 — 데스크톱 앱 (7~8주):**

| 주차 | 단계 | 산출물 |
|------|------|--------|
| 7주 | Tauri 셋업 + 코어 연동 | Tauri 프로젝트, TauriStorageAdapter, 글로벌 단축키, 클립보드 감시 |
| 8주 | 데스크톱 UI + 배포 | 플로팅 윈도우 UI, 시스템 트레이, GitHub Releases 배포 |

### 5.2 주차별 상세

**1주차 — 모노레포 + 코어 패키지**

- pnpm workspace 구성 (core / chrome-extension / desktop / shared-ui)
- @promptcraft/core: 어댑터 인터페이스 정의 (IStorageAdapter, IClipboardAdapter, IPlatformAdapter)
- @promptcraft/core: AnalyzerService, ProfileManager, ContextBuilder 스켈레톤
- @promptcraft/core: ApiClient (백엔드 호출)
- shared-ui: HighlightReport, MissingInfo, ProfileCard, ContextPreview 스켈레톤
- AWS Lambda + API Gateway 셋업

**2주차 — 크롬 확장 Shell**

- Vite + CRXJS + React + TypeScript 보일러플레이트
- Manifest V3 설정 (permissions: activeTab, storage, tabs)
- ChromeStorageAdapter 구현 (IStorageAdapter 구현체)
- 팝업 3페이지 라우팅 (분석 / 프로필 / 설정)
- 프로필 CRUD UI (생성/편집/삭제/활성화 토글)

**3주차 — 해석 투명화**

- AnalyzerService 로직 구현 (코어 패키지)
- Claude API 연동 (해석 분석 전용 시스템 프롬프트)
- 응답 JSON 파싱 및 3색 리포트 렌더링 (shared-ui)
- 프롬프트 직접 입력 모드
- 분석 프롬프트 반복 튜닝 (정확도 목표: 80%+)

**4주차 — 맥락 빌더**

- ProfileManager, ContextBuilder 로직 구현 (코어 패키지)
- Content Script: claude.ai 채팅 입력창 DOM 캡처
- 맥락 삽입 로직 (프로필 → 포맷팅 → 프롬프트 앞에 삽입)
- 팝업에서 삽입될 맥락 미리보기
- 단축키 설정 (Ctrl+Shift+P: 맥락 삽입 토글)

**5주차 — 기능 연동 + 확장**

- 해석 리포트의 "부재 정보" → 프로필 매칭 로직
- 자동 보충 가능 항목 표시 (✅ 아이콘)
- "보강된 프롬프트" 생성 및 복사/전송 기능
- Content Script: chatgpt.com 채팅 입력창 대응
- 에러 핸들링, 수동 테스트 시나리오 20개

**6주차 — 크롬 확장 베타**

- Chrome Web Store 개발자 등록 및 제출
- 랜딩 페이지 간단 제작
- 초기 사용자 10~20명 모집 (개발자 커뮤니티)
- 피드백 수집 (사용 로그 + 간단 설문)

**7주차 — 데스크톱 앱 셋업**

- Tauri 2.0 프로젝트 생성
- TauriStorageAdapter 구현 (SQLite 기반, IStorageAdapter 구현체)
- TauriClipboardAdapter 구현 (클립보드 감시)
- 글로벌 단축키 등록 (Ctrl+Shift+A: 분석 트리거)
- 코어 패키지 + shared-ui 연동 확인

**8주차 — 데스크톱 UI + 배포**

- 플로팅 윈도우 UI (분석 결과 표시)
- 시스템 트레이 아이콘 + 메뉴 (프로필 전환, 설정)
- Windows / macOS 빌드 테스트
- GitHub Releases 배포 (자동 업데이터 설정)
- 크롬 확장 피드백 반영
