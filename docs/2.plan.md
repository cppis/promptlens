## 2. 기술 아키텍처

### 2.1 아키텍처

```
Claude Desktop / Claude Code
    │
    │  MCP (stdio, JSON-RPC)
    │
    ▼
┌──────────────────────────────────┐
│  PromptLens MCP Server          │
│                                  │
│  index.js (도구 등록 + 라우팅)    │
│    │                             │
│    ├── analyzer.js               │
│    │   ├── analyzePrompt()       │  ← local 모드 (규칙 엔진)
│    │   └── analyzePromptWithApi()│  ← api 모드 (Claude API)
│    │                             │
│    ├── storage.js                │
│    │   ├── data.json 관리        │  ← 프로젝트 + 히스토리
│    │   └── settings.json 관리    │  ← API 키 + 모델
│    │                             │
│    └── importer.js               │
│        ├── importClaudeDesktop() │  ← .json
│        └── importClaudeCode()    │  ← .jsonl
│                                  │
│  ~/.promptlens/                 │
│    data.json                     │
│    settings.json                 │
└──────────────────────────────────┘
         │
         │ HTTPS (api 모드만, 사용자 API 키)
         ▼
    Claude API (Anthropic)
```

### 2.2 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|---------|
| MCP 런타임 | @modelcontextprotocol/sdk | 공식 SDK, stdio transport |
| 스키마 검증 | Zod | 도구 파라미터 타입 안전성 |
| AI 분석 | Claude API (fetch) | BYOK, 서버 불필요 |
| 저장소 | JSON 파일 (fs) | 의존성 없음, 단순, 백업 용이 |
| 테스트 | node:test (내장) | 외부 의존성 없음 |
| 런타임 | Node.js 18+ | ES Modules, 내장 fetch |

### 2.4 데이터 모델

**HistoryEntry:**

```javascript
{
  id:          string,     // h_timestamp_random
  projectId:   string,
  prompt:      string,     // 원본 프롬프트
  enhanced:    string,     // 보강된 프롬프트
  score:       number,     // 0-100
  axisScores:  number[],   // [clarity, specificity, context, structure, actionability]
  tags:        string[],
  note:        string,
  platform:    string,     // "claude"
  createdAt:   string,     // ISO 8601
  updatedAt:   string
}
```

**Settings:**

```javascript
{
  apiKey:   string,   // Anthropic API 키 (로컬 저장)
  model:    string    // 기본: claude-sonnet-4-5-20250514
}
```

## 3. v0.4.0 개발 계획

### 3.1 목표

두 가지 신규 기능으로 PromptLens의 핵심 가치를 강화한다.

| 기능 | 목표 | 핵심 가치 |
|------|------|----------|
| AI 해석 시뮬레이션 | AI 1인칭 서술로 해석 과정 체험 | 진입 장벽 최소화, "아하 모먼트" |
| 누적 패턴 코칭 | 반복 약점 추출 + 개인화 행동 제안 | 일회성 → 지속적 성장 도구 |

### 3.2 구현 일정 (5주)

| 주차 | 단계 | 산출물 |
|------|------|--------|
| 1주 | AI 해석 시뮬레이션 — local 모드 | analyzer.js에 narrative 생성 로직 (템플릿 기반 서술문), analyze_prompt 응답에 narrative 필드, HistoryEntry에 narrative 저장 |
| 2주 | AI 해석 시뮬레이션 — api 모드 | Claude API 시스템 프롬프트 확장 (1인칭 서술 지시), api 응답에서 narrative 파싱, local/api 서술 품질 비교 테스트 |
| 3주 | 누적 패턴 코칭 — 분석 엔진 | coaching.js 신규 모듈 (축별 통계, Missing 패턴 빈도, 도메인별 교차분석, 성장 추이, 코칭 메시지 템플릿) |
| 4주 | 누적 패턴 코칭 — MCP 도구 | get_coaching 도구 등록 (period, projectId), 최소 데이터 검증 (10개 미만 안내), 테스트 코드 |
| 5주 | 통합 테스트 + 문서 | narrative + coaching E2E 테스트, 문서 업데이트, v0.4.0 릴리스 |

### 3.3 도구 변경

| 유형 | 도구 | 내용 |
|------|------|------|
| 수정 | `analyze_prompt` | 응답에 `narrative` 필드 추가 |
| 신규 | `get_coaching` | 누적 패턴 분석 + 개인화 코칭 리포트 |

도구 총 수: 9개 → **10개**

### 3.4 파일 변경

```
mcp-server/
├── index.js                      ← get_coaching 도구 등록 추가
├── lib/
│   ├── analyzer.js               ← narrative 생성 로직 추가
│   ├── coaching.js               ← [신규] 패턴 분석 + 코칭 메시지
│   ├── storage.js                ← (변경 없음)
│   └── importer.js               ← (변경 없음)
├── test/
│   ├── analyzer.test.js          ← narrative 테스트 추가
│   ├── coaching.test.js          ← [신규]
│   └── mcp-tools.test.js         ← get_coaching 통합 테스트 추가
└── package.json                  ← v0.4.0
```
