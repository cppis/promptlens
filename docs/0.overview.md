# PromptLens — 프로젝트 개요

> "AI가 내 프롬프트를 어떻게 읽었는지 보여주고, 반복되는 약점을 코칭해준다."

---

## 1. PromptLens란?

PromptLens는 사용자의 프롬프트를 AI가 어떻게 해석하는지 **명시적으로 보여주는** MCP(Model Context Protocol) 도구다. Claude Desktop/Code에서 자연어로 호출하면 프롬프트의 품질 점수, 누락 요소, 개선 제안을 즉시 받을 수 있다.

핵심은 "점수 매기기"가 아니라 **AI의 해석 과정을 투명하게 드러내는 것**이다. AI가 프롬프트에서 무엇을 이해했고, 무엇을 추측했고, 무엇을 모르는지를 보여주면 사용자는 자연스럽게 더 나은 프롬프트를 쓰게 된다.

## 2. 핵심 기능

### 2.1 프롬프트 분석 (v0.3.0 — 현재)

두 가지 분석 모드를 제공한다.

**local 모드 (기본, 무료):** 규칙 엔진 기반으로 5축 점수(Clarity, Specificity, Context, Structure, Actionability)와 누락 요소, 개선 제안을 즉시 산출한다. API 키 불필요.

**api 모드 (선택, BYOK):** Claude API를 호출하여 3색 리포트를 생성한다.

| 색상 | 의미 | 예시 |
|------|------|------|
| 🟢 Referenced | AI가 프롬프트에서 명확히 이해한 것 | "React" → JS UI 라이브러리로 해석 |
| 🟡 Inferred | AI가 명시받지 않았지만 추론한 것 | "웹 앱" → React 언급에서 추론 |
| 🔴 Missing | AI가 모르는 정보, 모호한 부분 | "인증 방식" → 이메일? OAuth? SSO? |

### 2.2 AI 해석 시뮬레이션 (v0.4.0 — 예정)

3색 분류 목록을 넘어, AI가 1인칭 시점으로 해석 과정을 서술한다.

```
"나는 이 프롬프트를 받고 REST API 설계를 원한다고 이해했다.
다만 시스템 규모를 명시하지 않아서 중소규모로 가정했고,
출력 형식이 없으므로 자유 서술로 답하겠다."
```

점수는 "무엇이 부족한지"를 알려주지만, 해석 시뮬레이션은 **"AI가 어떻게 오해하는지"**를 보여준다. 프롬프팅 지식이 없는 사람도 즉시 이해할 수 있다.

### 2.3 누적 패턴 코칭 (v0.4.0 — 예정)

분석 이력에서 반복되는 약점 패턴을 추출하고 개인화된 코칭을 제공한다.

```
"이번 주 23개 분석 중 18개에서 출력 형식이 빠졌습니다.
코드 리뷰 프롬프트는 Context가 강하지만 Format이 약합니다.
다음 프롬프트에 '출력 형식: 표 형태로...'를 추가해보세요."
```

일회성 분석 도구를 **지속적 성장 도구**로 전환한다.

### 2.4 프로젝트 관리 & 히스토리

프롬프트를 프로젝트별로 묶어 관리하고, 분석 이력을 추적한다. Claude Desktop/Code 대화를 Import하여 기존 프롬프트를 일괄 분석할 수 있다.

## 3. 아키텍처

```
Claude Desktop / Claude Code
    │
    │  MCP (stdio)
    │
    ▼
┌──────────────────────────────────┐
│  PromptLens MCP Server (v0.3.0) │
│                                  │
│  ┌────────────┐  ┌────────────┐ │
│  │ Local 분석  │  │ API 분석   │ │
│  │ (규칙 엔진) │  │ (Claude)   │ │
│  └──────┬─────┘  └──────┬─────┘ │
│         │               │       │
│  ┌──────┴───────────────┘       │
│  │                              │
│  ▼                              │
│  ~/.promptlens/                │
│    data.json (프로젝트/히스토리)  │
│    settings.json (API 키/모델)   │
└──────────────────────────────────┘
```

**MCP Server가 유일한 클라이언트.** UI 없이 Claude Desktop/Code의 대화 인터페이스에서 자연어로 모든 기능을 사용한다.

## 4. MCP 도구

| 도구 | 설명 | 버전 |
|------|------|------|
| `analyze_prompt` | 프롬프트 품질 분석 (local/api 모드) | v0.3.0 |
| `list_projects` | 프로젝트 목록 + 통계 | v0.3.0 |
| `create_project` | 프로젝트 생성 | v0.3.0 |
| `get_history` | 분석 이력 조회 | v0.3.0 |
| `add_history_entry` | 히스토리에 프롬프트 추가 | v0.3.0 |
| `import_claude_conversations` | Claude 대화 Import | v0.3.0 |
| `get_stats` | 전체 통계 | v0.3.0 |
| `set_api_key` | API 키 설정 | v0.3.0 |
| `get_settings` | 설정 조회 | v0.3.0 |
| `get_coaching` | 누적 패턴 코칭 리포트 | v0.4.0 예정 |

## 5. 기술 스택

| 계층 | 기술 |
|------|------|
| MCP 런타임 | @modelcontextprotocol/sdk (stdio) |
| 스키마 검증 | Zod |
| AI 분석 | Claude API (BYOK, api 모드) |
| 저장소 | JSON 파일 (~/.promptlens/) |
| 테스트 | node:test (68개) |
| 런타임 | Node.js 18+ (ES Modules) |
