# Promptic — 프로젝트 개요

> "AI가 내 프롬프트를 어떻게 읽었는지 보여주고, 반복되는 약점을 코칭해준다."

---

## 1. Promptic 이란?

Promptic는 사용자의 프롬프트를 AI가 어떻게 해석하는지 **명시적으로 보여주는** MCP(Model Context Protocol) 도구다. Claude Desktop/Code에서 자연어로 호출하면 프롬프트의 품질 점수, 누락 요소, 개선 제안을 즉시 받을 수 있다.

핵심은 "점수 매기기"가 아니라 **AI의 해석 과정을 투명하게 드러내는 것**이다. AI가 프롬프트에서 무엇을 이해했고, 무엇을 추측했고, 무엇을 모르는지를 보여주면 사용자는 자연스럽게 더 나은 프롬프트를 쓰게 된다.

## 2. 핵심 기능

### 2.1 프롬프트 분석 (v0.4.0 — 현재)

두 가지 분석 모드를 제공한다.

**local 모드 (기본, 무료):** 규칙 엔진 기반으로 5축 점수(Clarity, Specificity, Context, Structure, Actionability)와 누락 요소, 개선 제안을 즉시 산출한다. API 키 불필요.

**api 모드 (선택, BYOK):** Claude API를 호출하여 3색 리포트를 생성한다.

| 색상 | 의미 | 예시 |
|------|------|------|
| 🟢 Referenced | AI가 프롬프트에서 명확히 이해한 것 | "React" → JS UI 라이브러리로 해석 |
| 🟡 Inferred | AI가 명시받지 않았지만 추론한 것 | "웹 앱" → React 언급에서 추론 |
| 🔴 Missing | AI가 모르는 정보, 모호한 부분 | "인증 방식" → 이메일? OAuth? SSO? |

### 2.2 AI 해석 시뮬레이션 (v0.5.0 — 예정)

3색 분류 목록을 넘어, AI가 1인칭 시점으로 해석 과정을 서술한다.

```
"나는 이 프롬프트를 받고 REST API 설계를 원한다고 이해했다.
다만 시스템 규모를 명시하지 않아서 중소규모로 가정했고,
출력 형식이 없으므로 자유 서술로 답하겠다."
```

점수는 "무엇이 부족한지"를 알려주지만, 해석 시뮬레이션은 **"AI가 어떻게 오해하는지"**를 보여준다. 프롬프팅 지식이 없는 사람도 즉시 이해할 수 있다.

### 2.3 누적 패턴 코칭 (v0.5.0 — 예정)

분석 이력에서 반복되는 약점 패턴을 추출하고 개인화된 코칭을 제공한다.

```
"이번 주 23개 분석 중 18개에서 출력 형식이 빠졌습니다.
코드 리뷰 프롬프트는 Context가 강하지만 Format이 약합니다.
다음 프롬프트에 '출력 형식: 표 형태로...'를 추가해보세요."
```

일회성 분석 도구를 **지속적 성장 도구**로 전환한다.

### 2.4 버전 관리 & 비교 (v0.4.0 — 현재)

프롬프트를 개선할 때마다 `parentId`로 이전 버전에 연결하면 자동 버전 체인이 형성된다. `compare_prompts`로 두 버전 간 텍스트 diff, 점수 변화, 5축 변화를 한눈에 비교할 수 있고, `get_versions`로 전체 개선 이력을 추적한다.

### 2.5 대시보드 시각화 (v0.4.0 — 현재)

프로젝트의 분석 히스토리를 Chart.js 기반 HTML 대시보드로 시각화한다. 점수 추이(라인), 5축 레이더(최근 vs 평균), 등급 분포(도넛), 태그 통계(바), 최근 분석 테이블을 포함한다. `>> viz` 또는 `>> 시각화` 커맨드로 실행.

### 2.6 커맨드 패턴 & 자연어 자동 감지 (v0.4.0 — 현재)

도구 설명에 트리거 룰을 내장하여 Claude가 자동으로 Promptic를 호출한다. `>> anz`, `>> anz+run`, `>> deep`/`>> 정밀분석`, `>> viz`, `>> diff` 커맨드와 한국어/영어 자연어 패턴을 지원한다.

### 2.7 MCP Prompts 워크플로우 (v0.4.0 — 현재)

`server.prompt()`로 3개 워크플로우 프리셋을 제공한다. Claude Desktop `/` 메뉴에서 선택 가능: `quick-analyze`(빠른 분석), `deep-analyze`(4단계 정밀 분석 파이프라인), `project-report`(대시보드 생성).

### 2.8 프로젝트 관리 & 히스토리

프롬프트를 프로젝트별로 묶어 관리하고, 분석 이력을 추적한다. Claude Desktop/Code 대화를 Import하여 기존 프롬프트를 일괄 분석할 수 있다.

### 2.9 로컬 파일 저장소 & 고급 조회 (v0.4.0 — 현재)

프로젝트 데이터를 프로젝트별 독립 파일(`~/.promptic/projects/{id}.json`)로 분리 저장한다. 단일 `data.json`의 비대화 문제를 해결하고, 팀 공유와 고급 쿼리가 가능해진다.

`export_project`로 JSON/Markdown/CSV 형식의 로컬 파일을 생성하고, `import_project`로 다른 머신에서 복원할 수 있다. `query_history`는 점수 범위, 등급, 날짜, 태그, 키워드 조건을 조합한 필터링과 정렬을 지원한다. `snapshot_project`는 특정 시점의 프로젝트 상태를 저장하고 두 스냅샷 간 diff를 제공한다.

## 3. 아키텍처

```
Claude Desktop / Claude Code
    │
    │  MCP (stdio)
    │
    ▼
┌──────────────────────────────────────────────┐
│  Promptic MCP Server (v0.4.0)              │
│                                              │
│  ┌────────────┐  ┌────────────┐              │
│  │ Local 분석  │  │ API 분석   │              │
│  │ (규칙 엔진) │  │ (Claude)   │              │
│  └──────┬─────┘  └──────┬─────┘              │
│         │               │                    │
│  ┌──────┴───────────────┘                    │
│  │                                           │
│  ├── Differ   (word-level diff, LCS)         │
│  ├── Visualizer (Chart.js HTML 대시보드)      │
│  ├── Exporter  (JSON / Markdown / CSV)       │
│  ├── Query     (필터 + 정렬 엔진)             │
│  │                                           │
│  ▼                                           │
│  ~/.promptic/                              │
│    projects/                                 │
│      _index.json          (프로젝트 목록)     │
│      {projectId}.json     (프로젝트+히스토리) │
│    exports/               (내보낸 파일)       │
│    snapshots/             (스냅샷 파일)       │
│    settings.json          (API 키/모델)      │
│    dashboard-*.html       (시각화 리포트)     │
└──────────────────────────────────────────────┘
```

**MCP Server가 유일한 클라이언트.** UI 없이 Claude Desktop/Code의 대화 인터페이스에서 자연어로 모든 기능을 사용한다.
